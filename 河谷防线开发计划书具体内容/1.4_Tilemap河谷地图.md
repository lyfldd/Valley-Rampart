# 1.4 河谷地形

### 优先级：最高

---

## 设计理念

每种地形**一张复合大图**——天空+远景+地表+剖面全在一张 PNG 里。地图由多张地形图横向拼接组成，不再用多层 Tilemap 拼装。

```
Y轴 ┌─────────────────────────────┐
    │           天空                │
    │  地形对应远景（自带）           │
    ├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
    │  玩家行走区                   │  ← 全部在地表线之上
    │═══ 地表线 ═══════════════════│  ← 玩家脚下
    │  地下剖面（自带）              │  ← 不可交互，纯视觉
    └─────────────────────────────┘
```

## 7 种地形（无等级系统）

| # | 地形 | 资源 | 可耕种 | 备注 |
|:-:|------|:----:|:------:|------|
| 1 | 平原 (Plains) | 无 | ✅ 1× 粮食 | 默认地形 |
| 2 | 肥沃 (Fertile) | 无 | ✅ 2× 粮食 | 耕种升级，产粮更快 |
| 3 | 荒地 (Barren) | 无 | ❌ | 沙地荒原，不可耕种 |
| 4 | 丘陵 (Hills) | 无 | ❌ | 起伏地形，可建造 |
| 5 | 林地 (Forest) | 木材 | ❌ | 产木材 |
| 6 | 矿山 (Mine) | 石材+矿石 | ❌ | 采石场/矿场 |
| 7 | 雪山 (Snow Mountain) | 无 | ❌ | 寒冷地形，移动慢 |

> **取消**：等级系统（1-3级）、河流系统、背景决定等级的设定。

## 资源-地形-生产逻辑

**地形只管"能造什么"，不管"产多少"。产量由建造系统的建筑等级决定。**

```
地形    →  允许建造               →  建筑等级 → 实际产量
────────────────────────────────────────────────────────
林地    →  伐木场                  →  Lv1/2/3  → 木材 × N
矿山    →  采石场 / 矿场           →  Lv1/2/3  → 石材 × N / 矿石 × N
平原    →  农田                    →  Lv1/2/3  → 粮食 × N × 1
肥沃    →  农田                    →  Lv1/2/3  → 粮食 × N × 2
荒地    →  无生产建筑              →  —        →  无
丘陵    →  无生产建筑              →  —        →  无
雪山    →  无生产建筑              →  —        →  无
```

> 肥沃是唯一的例外：地形本身提供 2× 倍率，但基础产量仍由农田等级决定。
> 建筑等级和产量数值由阶段2 `BuildingData` 配置，不在地形系统里。

## 场景结构

```
Main.unity
└── Grid (Cell Size: 1,1,0)
    ├── TerrainLayers  (Tilemap + TilemapRenderer, Order: 0)
    │      ↓ 7 种地形图，按设计在水平方向拼接
    └── Decoration     (独立 SpriteRenderer, Order: 1)
           ↓ 建筑/树木/玩家/单位
```

> 单一 Tilemap 负责铺贴地形图；建筑和单位用 SpriteRenderer 独立管理。

## 数据模型

```csharp
public enum TerrainType
{
    Plains,     // 平原
    Fertile,    // 肥沃
    Barren,     // 荒地
    Hills,      // 丘陵
    Forest,     // 林地
    Mine,       // 矿山
    SnowMountain// 雪山
}

public enum ResourceType
{
    None,
    Wood,       // 木材 — 林地
    Stone,      // 石材 — 矿山
    Ore,        // 矿石 — 矿山
    Food,       // 粮食 — 平原/肥沃
    Gold        // 金币 — NPC
}

[CreateAssetMenu(menuName = "ValleyRampart/TerrainTileData")]
public class TerrainTileData : ScriptableObject
{
    [Header("基本信息")]
    public TerrainType terrainType;
    public string displayName;
    public Sprite compositeSprite;    // 整张地形图（天空+远景+地表+剖面）

    [Header("资源")]
    public ResourceType resourceType;   // 地形能产什么（None = 不产出）
    public bool canFarm = false;        // 可否耕种
    public float farmMultiplier = 1f;   // 耕种产出倍率（肥沃=2，平原地形本身=1）

    [Header("玩法")]
    public bool canBuild = true;        // 可否建造
    public float moveSpeedModifier = 1f; // 雪山=0.7 等
}
```

## 实现步骤

### 第一期（Day 1-2）

1. 创建 `TerrainTileData.cs` ScriptableObject
2. 在 `Assets/_Game/Config/Terrain/` 创建 7 个 SO 实例，挂上对应图片
3. 创建场景结构（Grid + 1 个 Tilemap + Decoration 节点）
4. 编写 `TerrainMapInitializer.cs`：根据地图数据在 Tilemap 上按位置铺对应地形图
5. 绘制河谷地图（横向 300 tile = 9600px），按设计从左到右铺地形
6. 主城放在 tile ~100 位置（平原或肥沃）

### 第二期（阶段2）

- 接 SeasonManager（雪山的季节视觉调整）
- 接资源管理器（林地/矿山的自动采集）
- 视差滚动（地形图自身视差）
