# 1.3 EventBus（泛型事件系统）

### 优先级：🔴 最高（所有系统都依赖它）

### 脚本

| 文件 | 路径 | 说明 |
|------|------|------|
| `EventBus.cs` | `Assets/_Game/Core/` | 泛型事件总线，静态类 |
| `GameEvents.cs` | `Assets/_Game/Core/` | 所有事件 struct 定义 |

### EventBus 核心设计

```
EventBus 是静态泛型类：
  - 内部维护 Dictionary<Type, Delegate> 订阅表
  - Subscribe<T>(Action<T> handler)   → 注册监听
  - Unsubscribe<T>(Action<T> handler) → 移除监听
  - Publish<T>(T eventData)           → 广播事件（struct 传值，无 GC）
```

### 关键实现要点

1. **使用 `struct` 定义事件，不用 `class`**——避免 GC 分配
2. **所有事件 struct 标记 `readonly`**——防止误改
3. **Publish 时遍历 Delegate.GetInvocationList()**——逐个调用，某个 handler 抛异常不影响其他
4. **不需要** `MonoBehaviour` 依赖——纯 C# 静态类

| `TerrainModifiedEvent` | `Vector2Int tilePos, int newTileType` | TerrainModification | EnemyPathfinding |
| `DefenseLineBreachedEvent` | `int lineIndex` (0壕沟1外墙2内墙) | CombatSystem | UI, RulerSkill, FactionSystem |
| `WaveCompletedEvent` | `int waveIndex` | WaveManager | ResourceManager, UI, FactionSystem |
| `FactionRelationChangedEvent` | `int factionId, float newRelation` | DiplomacyManager | UI, WaveSpawner, ExpeditionManager |

| `RulerSkillUsedEvent` | `int skillId` | RulerSkillSystem | VFX, Audio, UI |

| `ExpeditionStartedEvent` | `int targetIslandId` | ExpeditionManager | UI, SaveSystem |
| `GameOverEvent` | `bool isVictory` | CombatSystem | UI, SaveSystem |

### 验证标准
- [ ] 新建空场景，挂一个测试脚本：Subscribe → Publish → 回调被调用
- [ ] 同一个事件多个订阅者，全部收到
- [ ] Unsubscribe 后不再收到
- [ ] Publish 时其中一个 handler 抛异常，其他 handler 正常执行
