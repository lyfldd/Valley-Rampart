# 河谷防线 · 存档系统设计文档

> 版本：v1.0 · 2026-07-21
> 状态：**已被 v2 部分替代**——`SaveManager` 的硬编码 `CaptureSnapshot` / `ApplySnapshot` 部分改用 ISaveable 架构，见 [`存档系统设计文档_v2_ISaveable架构.md`](./存档系统设计文档_v2_ISaveable架构.md)
> 本文档仍有效的部分：POCO 数据结构、原子写入、版本迁移、命名重构建议（`UnitDataManager` → `UnitConfigCatalog`）

---

## 0. TL;DR

- 你现在的 `UnitDataManager` **名字起错了**——它加载的是「配置模板」（ScriptableObject），不是「数据」。建议改名 `UnitConfigCatalog`，定位是只读静态配置。
- 存档要管的是「运行时数据」：血量、位置、资源、天数。这套东西得从零建，不能复用 `UnitDataManager`。
- 推荐三层结构：**存档数据层（POCO）→ 存档管理器（SaveManager）→ 序列化器（JSON）**。
- 加载存档 ≠ 加载配置。加载存档是「读 JSON → 重建场景 → 用存档值覆盖默认值」的过程。

## 1. 先泼冷水：当前设计的问题

### 1.1 `UnitDataManager` 名不副实

现在的实现：

```csharp
public class UnitDataManager : Singleton<UnitDataManager>
{
    // 加载 Resources/UnitData/ 下所有 UnitData SO
    public void LoadAll() { ... }
}
```

它干的事：**把 `.asset` 配置文件搬进内存字典，按 `faction_occupation` 查表**。

这是典型的 **Config Catalog（配置目录）** 模式，不是 Data Manager。真正的「数据管理」应该管运行时状态——比如这个单位还剩多少血、站在哪、是不是死了。

**问题：**

1. **误导命名**——`UnitDataManager` 听起来像「管单位数据的」，新人（包括三个月后的你）会以为存档该往这塞，结果它只认只读 SO。
2. **职责单一但名字过大**——它只做一件事（查表），却顶着 `Data` 这种宽泛的名字。
3. **和存档系统概念冲突**——`UnitData` 这个类名同时被两处用：SO 配置文件叫 `UnitData`，运行时状态也该叫 `UnitData`？撞名了。

**建议：**

| 现状 | 改成 | 职责 |
|---|---|---|
| `UnitDataManager` | `UnitConfigCatalog` | 只读 SO 查表，「配置目录」 |
| `UnitData`（SO） | `UnitConfig` | 静态模板，「配置」 |
| （新增） | `UnitSaveData` | 运行时快照，「存档数据」 |

> 命名原则：**`Config` = 静态模板（策划填的数值）；`SaveData` = 运行时快照（玩家玩出来的状态）。**

### 1.2 「加载器」概念混淆

你说：「我们现在有很多的加载器都是去加载 asset 文件去初始化……但是我想要现在去加载一个存档的功能」。

这两种「加载」完全不同：

| | 加载配置（Asset Loading） | 加载存档（Save Loading） |
|---|---|---|
| **时机** | 游戏启动时，一次性 | 玩家点「继续游戏」时 |
| **来源** | `Resources/` 下的只读 `.asset` | `persistentDataPath/` 下的 JSON 文件 |
| **结果** | 得到一份「默认模板」 | 得到一份「玩家进度快照」 |
| **可写** | 只读 | 可读可写 |

`UnitDataManager.LoadAll()` 是前者。存档系统是后者。**两者不冲突，但要分开。**

## 2. 存档系统三层架构

```
┌─────────────────────────────────────────┐
│  SaveManager（MonoBehaviour 单例）        │
│  Save(slotId) / Load(slotId) / Delete()  │
│  协调：抓快照 → 序列化 → 写文件           │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│  GameSaveData（纯 C# POCO）              │
│  [Serializable] class，不继承任何东西     │
│  只有字段，没有方法，JsonUtility 可序列化  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│  JsonSaveSerializer                      │
│  Serialize(GameSaveData) → string        │
│  Deserialize(string) → GameSaveData      │
│  版本号处理、迁移钩子                      │
└─────────────────────────────────────────┘
```

**核心原则：存档数据层（POCO）不依赖 UnityEngine。** 这样理论上可以脱离 Unity 测试序列化逻辑（虽然实际开发中你大概率不会这么做）。

## 3. 存档数据层（POCO）设计

### 3.1 需要存什么？

从当前代码盘一遍运行时状态：

| 来源 | 运行时字段 | 要不要存 |
|---|---|---|
| `UnitController` | `CurrentHp`, `MaxHp`, `Attack`, `Defense`, `WalkSpeed`, `RunSpeed`, `transform.position` | ✅ 必存 |
| `RulerController` | `Gold`, `Stone`, `Wood`, `Food` | ✅ 必存 |
| `TimeManager` | `CurrentDay`, `CurrentTimeOfDay`, `CurrentSeason`, `CurrentPhase`, `secondsPerDay`, `totalDays`, `daysPerSeason` | ✅ 必存 |
| `UnitRegistry` | 所有存活单位列表 | ✅ 派生自单位列表，不单独存 |
| `GameStateManager` | `CurrentState` | ❌ 不存（读档后总是从 `Ready` 开始） |
| `UnitData`（SO 引用） | `faction`, `occupation` | ⚠️ 存枚举值，不存 SO 引用 |

### 3.2 存档数据结构

```csharp
using System;
using System.Collections.Generic;

/// <summary>
/// 单个单位的存档数据。纯 C#，不继承 MonoBehaviour。
/// 注意：字段必须是 public 或标 [SerializeField]，JsonUtility 才能序列化。
/// </summary>
[Serializable]
public class UnitSaveData
{
    // —— 身份（用于重建时查配置）——
    public int faction;        // Faction 枚举转 int
    public int occupation;     // Occupation 枚举转 int

    // —— 运行时状态 ——
    public int currentHp;
    public int maxHp;
    public int attack;
    public int defense;
    public float walkSpeed;
    public float runSpeed;

    // —— 位置（JsonUtility 不支持 Vector3，拆成基本类型）——
    public float posX;
    public float posY;

    // 预留：未来加 Buff/装备/状态效果
    // public List<string> activeBuffIds;
}

/// <summary>
/// 君主/国家资源存档。
/// </summary>
[Serializable]
public class RulerSaveData
{
    public int gold;
    public int stone;
    public int wood;
    public int food;

    // 君主单位本身的数据在 units 列表里（occupation == Ruler），不重复存
}

/// <summary>
/// 时间系统存档。
/// </summary>
[Serializable]
public class TimeSaveData
{
    public int currentDay;
    public float currentTimeOfDay;  // 0-24
    public int currentSeason;       // Season 枚举转 int
    public int currentPhase;        // TimePhase 枚举转 int

    // 配置项也要存，防止玩家中途改难度
    public float secondsPerDay;
    public int totalDays;
    public int daysPerSeason;
}

/// <summary>
/// 顶层存档容器。一个存档槽 = 一个 GameSaveData 实例 = 一个 JSON 文件。
/// </summary>
[Serializable]
public class GameSaveData
{
    // —— 元数据 ——
    public int saveVersion = 1;          // 存档格式版本号，用于迁移
    public string saveTime;              // ISO8601 时间戳，UI 显示用
    public string slotName;              // 槽位名（如 "slot_1"）

    // —— 游戏数据 ——
    public RulerSaveData ruler = new RulerSaveData();
    public TimeSaveData time = new TimeSaveData();
    public List<UnitSaveData> units = new List<UnitSaveData>();

    // 预留：未来加地图状态、建筑、任务进度
    // public MapSaveData map;
    // public List<BuildingSaveData> buildings;
}
```

**关键决策：**

1. **不存 SO 引用**——`UnitData` 是 `ScriptableObject`，`JsonUtility` 序列化不了 Unity 对象引用。存 `faction` + `occupation` 枚举值，读档时用 `UnitConfigCatalog.GetData(faction, occupation)` 反查配置。
2. **Vector3 拆开**——`JsonUtility` 不支持 `Vector3`/`Quaternion`，拆成 `float posX, posY`。如果未来要存 `Vector3`，写个 `[Serializable] struct Vec3 { float x, y, z; }`。
3. **枚举存 int**——`JsonUtility` 支持枚举，但存 int 更稳（防止未来改枚举名导致存档报废）。

## 4. 存档管理器（SaveManager）

### 4.1 职责

- **保存**：从各系统抓运行时快照 → 填充 `GameSaveData` → 序列化 → 写文件
- **加载**：读文件 → 反序列化 → 清空当前场景 → 重建单位 → 恢复状态
- **删除**：删文件
- **版本迁移**：检测到旧版本存档时，逐版本升级

### 4.2 接口设计

```csharp
using System;
using System.IO;
using UnityEngine;

/// <summary>
/// 存档管理器。负责保存/加载/删除存档，协调各系统的快照抓取与恢复。
/// </summary>
public class SaveManager : Singleton<SaveManager>
{
    private const int CurrentSaveVersion = 1;
    private const string SaveFolderName = "Saves";
    private const string SaveFileExtension = ".json";

    private readonly JsonSaveSerializer _serializer = new JsonSaveSerializer();

    // ===== 公共接口 =====

    /// <summary>保存当前游戏到指定槽位。</summary>
    public bool Save(string slotId)
    {
        try
        {
            GameSaveData data = CaptureSnapshot();
            data.saveVersion = CurrentSaveVersion;
            data.saveTime = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
            data.slotName = slotId;

            string json = _serializer.Serialize(data);
            string path = GetSavePath(slotId);

            // 原子写入：先写临时文件，再替换目标文件，防止写入中断导致存档损坏
            string tempPath = path + ".tmp";
            File.WriteAllText(tempPath, json);

            if (File.Exists(path)) File.Delete(path);
            File.Move(tempPath, path);

            Debug.Log($"[SaveManager] 保存成功: {path}");
            EventBus.Publish(new GameSavedEvent(slotId, true));
            return true;
        }
        catch (Exception e)
        {
            Debug.LogError($"[SaveManager] 保存失败: {e.Message}");
            EventBus.Publish(new GameSavedEvent(slotId, false));
            return false;
        }
    }

    /// <summary>从指定槽位加载存档。</summary>
    public bool Load(string slotId)
    {
        string path = GetSavePath(slotId);
        if (!File.Exists(path))
        {
            Debug.LogWarning($"[SaveManager] 存档不存在: {path}");
            return false;
        }

        try
        {
            string json = File.ReadAllText(path);
            GameSaveData data = _serializer.Deserialize(json);

            // 版本迁移
            if (data.saveVersion < CurrentSaveVersion)
            {
                data = MigrateSave(data);
            }

            ApplySnapshot(data);
            Debug.Log($"[SaveManager] 加载成功: {path}");
            EventBus.Publish(new GameLoadedEvent(slotId, true));
            return true;
        }
        catch (Exception e)
        {
            Debug.LogError($"[SaveManager] 加载失败: {e.Message}");
            EventBus.Publish(new GameLoadedEvent(slotId, false));
            return false;
        }
    }

    /// <summary>删除指定槽位的存档。</summary>
    public bool Delete(string slotId)
    {
        string path = GetSavePath(slotId);
        if (!File.Exists(path)) return false;

        File.Delete(path);
        Debug.Log($"[SaveManager] 删除存档: {path}");
        return true;
    }

    /// <summary>检查槽位是否有存档。</summary>
    public bool HasSave(string slotId)
    {
        return File.Exists(GetSavePath(slotId));
    }

    /// <summary>获取存档元数据（不加载完整数据，用于存档列表 UI）。</summary>
    public GameSaveData GetSaveMeta(string slotId)
    {
        string path = GetSavePath(slotId);
        if (!File.Exists(path)) return null;

        try
        {
            string json = File.ReadAllText(path);
            return _serializer.Deserialize(json); // 只读元数据，不 Apply
        }
        catch
        {
            return null;
        }
    }

    // ===== 快照抓取（保存时调用）=====

    private GameSaveData CaptureSnapshot()
    {
        var data = new GameSaveData();

        // 1. 抓君主资源
        var ruler = RulerController.Instance;
        data.ruler.gold = ruler.Gold;
        data.ruler.stone = ruler.Stone;
        data.ruler.wood = ruler.Wood;
        data.ruler.food = ruler.Food;

        // 2. 抓时间
        var time = TimeManager.Instance;
        data.time.currentDay = time.CurrentDay;
        data.time.currentTimeOfDay = time.CurrentTimeOfDay;
        data.time.currentSeason = (int)time.CurrentSeason;
        data.time.currentPhase = (int)time.CurrentPhase;
        data.time.secondsPerDay = time.SecondsPerDay;
        data.time.totalDays = time.TotalDays;
        data.time.daysPerSeason = time.DaysPerSeason;

        // 3. 抓所有存活单位
        foreach (var unit in UnitRegistry.Instance.GetAllUnits())
        {
            if (unit.Data == null) continue;

            var unitData = new UnitSaveData
            {
                faction = (int)unit.Data.faction,
                occupation = (int)unit.Data.occupation,
                currentHp = unit.CurrentHp,
                maxHp = unit.MaxHp,
                attack = unit.Attack,
                defense = unit.Defense,
                walkSpeed = unit.WalkSpeed,
                runSpeed = unit.RunSpeed,
                posX = unit.transform.position.x,
                posY = unit.transform.position.y
            };
            data.units.Add(unitData);
        }

        return data;
    }

    // ===== 快照应用（加载时调用）=====

    private void ApplySnapshot(GameSaveData data)
    {
        // 1. 清空当前场景（销毁所有单位）
        foreach (var unit in UnitRegistry.Instance.GetAllUnits())
        {
            if (unit != null) Destroy(unit.gameObject);
        }
        UnitRegistry.Instance.Clear();

        // 2. 恢复时间（先配置再状态，防止 AdvanceDay 误触发）
        var time = TimeManager.Instance;
        time.SetSecondsPerDay(data.time.secondsPerDay);
        time.SetTotalDays(data.time.totalDays);
        time.SetDaysPerSeason(data.time.daysPerSeason);
        // TimeManager 需要加一个 RestoreState 方法，直接设置 CurrentDay/CurrentTimeOfDay 等
        time.RestoreState(data.time.currentDay, data.time.currentTimeOfDay,
                          (Season)data.time.currentSeason, (TimePhase)data.time.currentPhase);

        // 3. 恢复君主资源
        var ruler = RulerController.Instance;
        ruler.RestoreResources(data.ruler.gold, data.ruler.stone, data.ruler.wood, data.ruler.food);

        // 4. 重建所有单位
        foreach (var unitData in data.units)
        {
            var faction = (Faction)unitData.faction;
            var occupation = (Occupation)unitData.occupation;

            // 从配置目录拿模板
            UnitData config = UnitDataManager.Instance.GetData(faction, occupation);
            if (config == null)
            {
                Debug.LogError($"[SaveManager] 找不到单位配置: {faction}_{occupation}，跳过");
                continue;
            }

            // 创建单位（会触发 Initialize，设置默认值）
            Vector2 pos = new Vector2(unitData.posX, unitData.posY);
            GameObject go = UnitFactory.Instance.SpawnUnit(config, pos);
            if (go == null) continue;

            // 用存档值覆盖默认值
            var controller = go.GetComponent<UnitController>();
            controller.RestoreFromSave(unitData);
        }

        Debug.Log($"[SaveManager] 场景重建完成，共恢复 {data.units.Count} 个单位");
    }

    // ===== 版本迁移 =====

    private GameSaveData MigrateSave(GameSaveData data)
    {
        // 逐版本升级：v1 → v2 → v3...
        // 每个迁移函数只负责升一级
        while (data.saveVersion < CurrentSaveVersion)
        {
            switch (data.saveVersion)
            {
                case 1:
                    // data = MigrateV1ToV2(data);
                    break;
                // case 2:
                //     data = MigrateV2ToV3(data);
                //     break;
                default:
                    Debug.LogError($"[SaveManager] 未知的存档版本: {data.saveVersion}");
                    return data;
            }
        }
        return data;
    }

    // ===== 工具 =====

    private string GetSavePath(string slotId)
    {
        string folder = Path.Combine(Application.persistentDataPath, SaveFolderName);
        if (!Directory.Exists(folder)) Directory.CreateDirectory(folder);
        return Path.Combine(folder, slotId + SaveFileExtension);
    }
}
```

/// <summary>从存档恢复时间状态。不发事件，直接设置。</summary>
public void RestoreState(int day, float timeOfDay, Season season, TimePhase phase)
{
    CurrentDay = day;
    CurrentTimeOfDay = timeOfDay;
    CurrentSeason = season;
    CurrentPhase = phase;
    CurrentHour = Mathf.Clamp(Mathf.FloorToInt(timeOfDay), 0, 23);
    _dayTimer = (timeOfDay / 24f) * secondsPerDay;

## 5. JSON 序列化器

```csharp
using UnityEngine;

/// <summary>
/// JSON 序列化器。封装 JsonUtility，未来可换成 Newtonsoft.Json / MessagePack。
/// </summary>
public class JsonSaveSerializer
{
    public string Serialize(GameSaveData data)
    {
        return JsonUtility.ToJson(data, prettyPrint: true);
    }

    public GameSaveData Deserialize(string json)
    {
        return JsonUtility.FromJson<GameSaveData>(json);
    }
}
```

**为什么单独抽一个类？** 如果未来要换序列化方案（比如存档加密、压缩、或换支持 `Dictionary` 的 Newtonsoft），只改这一个文件，不动 `SaveManager`。

## 7. 加载存档的完整流程

```
玩家点「继续游戏」
    │
    ▼
SaveManager.Load("slot_1")
    │
    ├─ 读 JSON 文件
    ├─ JsonUtility.FromJson → GameSaveData
    ├─ 版本检查 → 必要时迁移
    │
    ▼
ApplySnapshot(data)
    │
    ├─ 1. 清空场景（Destroy 所有单位）
    │
    ├─ 2. 恢复 TimeManager 状态
    │      └─ 调 RestoreState()，直接赋值，不发事件
    │
    ├─ 3. 恢复 RulerController 资源
    │      └─ 调 RestoreResources()
    │
    ├─ 4. 遍历 data.units
    │      │
    │      ├─ 用 faction+occupation 从 UnitConfigCatalog 查配置
    │      ├─ UnitFactory.SpawnUnit(config, pos)  ← 触发 Initialize，设置默认值
    │      └─ controller.RestoreFromSave(unitData)  ← 用存档值覆盖默认值
    │
    ▼
EventBus.Publish(new GameLoadedEvent(...))
    │
    ▼
GameStateManager.SetState(Playing)
```

**关键点：**

- **先 Spawn 再覆盖**——`SpawnUnit` 会走 `Initialize`，把 `CurrentHp = MaxHp` 等默认值设好。然后 `RestoreFromSave` 用存档值覆盖。这样即使存档里少了某个字段（比如旧版本存档没有 `walkSpeed`），也能用默认值兜底。
- **TimeManager 不发事件**——恢复时间时如果触发 `TimeDayChangedEvent`，可能导致敌人刷新逻辑误执行。所以 `RestoreState` 直接赋值，不走 `AdvanceDay`。

## 8. 命名重构建议（可选但推荐）

如果你认同「`UnitDataManager` 名不副实」的判断，重构步骤：

1. `UnitData.cs` → 类名改 `UnitConfig`，`[CreateAssetMenu(menuName = "ValleyRampart/UnitConfig")]`
2. `RulerData.cs` → 类名改 `RulerConfig`，继承 `UnitConfig`
3. `UnitDataManager` → `UnitConfigCatalog`
4. 所有引用处批量替换（`UnitData` → `UnitConfig`，`UnitDataManager` → `UnitConfigCatalog`）
5. 已有的 `.asset` 文件不用动，SO 实例和类名解耦

**注意：** 改类名后，Inspector 里已序列化的引用会丢失（因为 Unity 按类名匹配）。需要重新拖一次配置。如果项目里 SO 引用很多，可以写个 Editor 脚本批量修复，或者干脆趁项目还小赶紧改。

## 9. 实施步骤（按依赖排序）

| 步骤 | 内容 | 依赖 |
|---|---|---|
| 1 | 定义 `GameSaveData` / `UnitSaveData` / `RulerSaveData` / `TimeSaveData` | 无 |
| 2 | 写 `JsonSaveSerializer` | 1 |
| 3 | 给 `UnitController` / `RulerController` / `TimeManager` 加 `Restore*` 方法 | 无 |
| 4 | 写 `SaveManager`（保存/加载/删除/迁移） | 1, 2, 3 |
| 5 | 加 `GameSavedEvent` / `GameLoadedEvent` 到 GameEvents.cs | 无 |
| 6 | UI 接存档列表（显示 `saveTime`、槽位选择） | 4 |
| 7 | （可选）重命名 `UnitDataManager` → `UnitConfigCatalog` | 无 |
| 8 | （可选）加自动存档（每天开始时自动存到 `autosave` 槽） | 4 |

## 10. 常见坑

1. **JsonUtility 不支持的类型**：`Dictionary`、`Vector3`、`Quaternion`、多态（继承 `UnitSaveData` 的子类反序列化时会丢类型）。遇到这些要么拆字段，要么换 Newtonsoft.Json。
2. **存档版本不兼容**：改了 `GameSaveData` 结构（删字段、改类型）后，旧存档反序列化会出错。所以 `saveVersion` 从第一天就要加。
3. **加载时单位重复注册**：`SpawnUnit` → `Initialize` → `UnitRegistry.Register`，如果加载前没清空场景，会重复。所以 `ApplySnapshot` 第一步就是 `Destroy` + `Clear`。
4. **TimeManager 恢复时触发事件**：`RestoreState` 直接赋值，不调 `AdvanceDay`，否则可能触发敌人刷新/季节变化逻辑。
5. **写入中断**：直接 `File.WriteAllText` 如果中途断电，存档文件损坏。用「先写临时文件再替换」的原子写入。

## 11. 总结：你的设计对不对？

| 你的判断 | 我的结论 |
|---|---|
| 「先定义存档数据层（纯 C# 可序列化类）」 | ✅ 完全正确，这是第一步 |
| 「实现中心化存档管理器，包含保存、加载、删除、版本迁移」 | ✅ 完全正确，这就是 SaveManager 的职责 |
| 「UnitDataManager 真正的作用是去加载 JSON 数据」 | ❌ 不对。它加载的是只读配置（SO），不是存档。存档系统要新建一套，别往 UnitDataManager 里塞 |
| 「现在的加载器都是加载 asset 初始化，这是一个功能」 | ✅ 对，这是「配置加载」，和「存档加载」是两回事，并行存在 |

**一句话：UnitDataManager 管「模板」，SaveManager 管「进度」，两者各司其职。**
