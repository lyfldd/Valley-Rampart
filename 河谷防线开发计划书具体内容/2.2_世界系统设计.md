# 🌍 世界运行底层逻辑管理器设计（v4）

> 2026-07-23 阿铁拍板，主理人游承峰汇编
> 升级自 v3《难度系统设计》——难度只是世界规则的一个子集，统一成 WorldSystem

## 📝 升级动机

阿铁的核心洞察：**难度管理器、时间管理器、世界管理器……本质都是"世界运行底层逻辑"的子管理器**。配置数据散在多处硬编码，应该统一到一个 SO 里数据驱动。

| v3 的问题 | v4 的修正 |
|----------|----------|
| 只设计了 DifficultyManager，没考虑其他世界规则 | 统一成 `WorldSystem` 门面 + 子管理器 |
| 难度配置单独一个 SO | 聚合到 `WorldConfig` SO（含时间/难度/季节/经济子配置） |
| 硬编码点散落在 TimeManager / NewGameConfig / RulerController | 全部抽到 `WorldConfig`，Inspector 可调 |

## 二、架构设计：WorldSystem + WorldConfig

### 2.1 架构总图

```
┌─────────────────────────────────────────────────────────────┐
│                  配置层（SO，只读，Inspector 可调）            │
│  WorldConfig.asset                                          │
│    ├─ TimeConfig       (秒/天, 天/季, 黎明黄昏, 起始天/时)   │
│    ├─ DifficultyConfig (三档预设, 基础资源, 季节增长量)       │
│    ├─ SeasonConfig     (四季日出日落时刻表)                  │
│    └─ SaveConfig       (自动存档间隔)                        │
└────────────────────────────┬────────────────────────────────┘
                             │ Resources.Load
                             ▼
┌─────────────────────────────────────────────────────────────┐
│              门面层（WorldSystem，初始化协调者）               │
│  WorldSystem : Singleton<WorldSystem>, ISaveable            │
│    ├─ 持有 WorldConfig 引用                                  │
│    ├─ InitializeWorld(NewGameConfig) ← 新建游戏统一入口      │
│    ├─ ApplyConfigToSubsystems() ← 把配置分发给子管理器       │
│    └─ 存档 WorldSaveData（聚合世界级状态）                   │
└──────┬──────────────────┬───────────────────┬───────────────┘
       │                  │                   │
       ▼                  ▼                   ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────────┐
│ TimeManager  │  │ Difficulty   │  │ WorldManager     │
│ (子管理器)    │  │ Manager      │  │ (子管理器)        │
│              │  │ (子管理器)    │  │                  │
│ 从Config读   │  │ 从Config读   │  │ 从Config读       │
│ 时间规则     │  │ 难度规则     │  │ 地图配置         │
│ ISaveable    │  │ ISaveable    │  │ ISaveable        │
└──────────────┘  └──────────────┘  └──────────────────┘
       │                  │
       ▼                  ▼
┌──────────────────────────────────────────────┐
│            存档层（各子管理器各自 ISaveable）    │
│  TimeManager → TimeSaveData                  │
│  DifficultyManager → DifficultySaveData      │
│  WorldManager → WorldSaveData                │
│  SaveManager 自动收集，无需改动               │
└──────────────────────────────────────────────┘
```

### 2.2 设计原则

1. **配置统一**：所有世界规则集中到 `WorldConfig` 一个 SO，Inspector 可调，不改代码
2. **门面协调**：`WorldSystem` 只做初始化协调和配置分发，**不接管子管理器的运行时逻辑**（避免 God Object）
3. **子管理器独立**：`TimeManager` / `DifficultyManager` / `WorldManager` 保持独立 Singleton，各自管运行时和存档
4. **存档不聚合**：各子管理器各自实现 `ISaveable`，复用现有 `SaveManager` 架构，**不破坏已有存档系统**
5. **新建/读档分叉**：新建走 `WorldSystem.InitializeWorld()`，读档走各子管理器 `LoadState()`（ISaveable 自动恢复）

## 三、WorldConfig SO 完整定义

**文件**：`Assets/_Game/Data/WorldConfig.cs`
**资产实例**：`Assets/Resources/World/WorldConfig.asset`

```csharp
using System;
using UnityEngine;

/// <summary>
/// 世界运行底层逻辑的统一配置（ScriptableObject）。
/// 所有世界规则集中在此，Inspector 可调，数据驱动。
/// 子管理器（TimeManager/DifficultyManager/WorldManager）从这里读自己的配置。
/// </summary>
[CreateAssetMenu(menuName = "ValleyRampart/WorldConfig", fileName = "WorldConfig")]
public class WorldConfig : ScriptableObject
{
    [Header("时间规则")]
    public TimeConfig time;

    [Header("难度规则")]
    public DifficultyConfig difficulty;

    [Header("季节规则")]
    public SeasonConfig season;

    [Header("存档规则")]
    public SaveConfig save;
}

// ===== 子配置 =====

/// <summary>时间规则配置。</summary>
[Serializable]
public struct TimeConfig
{
    [Tooltip("现实多少秒 = 游戏内一天")]
    public float secondsPerDay;       // 默认 480（8分钟/天）

    [Tooltip("游戏开始是第几天")]
    public int startDay;              // 默认 1

    [Tooltip("游戏开始当天的起始时刻 0-24")]
    public float startHour;           // 默认 6（早上6点）

    [Tooltip("一个季节持续多少天")]
    public int daysPerSeason;         // 默认 10

    [Tooltip("黎明过渡时长（小时）")]
    public float dawnDuration;        // 默认 1

    [Tooltip("黄昏过渡时长（小时）")]
    public float duskDuration;        // 默认 1
}

/// <summary>难度规则配置。</summary>
[Serializable]
public struct DifficultyConfig
{
    [Tooltip("三档预设（索引 0/1/2 对应 Easy/Normal/Hard）")]
    public DifficultyPreset[] presets;  // 3 个

    [Tooltip("基础资源（最低难度满额，实际 = 基础 ÷ 难度系数）")]
    public int baseGold;              // 默认 200
    public int baseStone;
    public int baseWood;
    public int baseFood;

    [Tooltip("每季难度系数增加多少")]
    public float factorGrowthPerSeason;  // 默认 0.25

    /// <summary>按档位值获取预设（档位 1/2/3 → 索引 0/1/2）。</summary>
    public DifficultyPreset GetPreset(int difficultyValue)
    {
        int index = Mathf.Clamp(difficultyValue - 1, 0, presets.Length - 1);
        return presets[index];
    }

    /// <summary>按档位值算初始资源（基础 ÷ 系数）。</summary>
    public ResourcePack GetInitialResources(int difficultyValue)
    {
        return new ResourcePack
        {
            gold = Mathf.FloorToInt(baseGold / difficultyValue),
            stone = Mathf.FloorToInt(baseStone / difficultyValue),
            wood = Mathf.FloorToInt(baseWood / difficultyValue),
            food = Mathf.FloorToInt(baseFood / difficultyValue)
        };
    }
}

/// <summary>单档难度预设。</summary>
[Serializable]
public struct DifficultyPreset
{
    public string name;              // "Easy" / "Normal" / "Hard"
    public int difficultyValue;      // 1 / 2 / 3（= DifficultyFactor 初始值）
    [Tooltip("现实秒/天。Easy 节奏慢，Hard 节奏紧")]
    public float secondsPerDay;      // Easy 600 / Normal 480 / Hard 360
}

/// <summary>季节规则配置（四季日出日落）。</summary>
[Serializable]
public struct SeasonConfig
{
    [Tooltip("四季的日出日落时刻")]
    public SeasonSunData[] seasons;  // 4 个：春夏秋冬

    public SeasonSunData GetSeason(Season season)
    {
        int index = Mathf.Clamp((int)season, 0, seasons.Length - 1);
        return seasons[index];
    }
}

/// <summary>单个季节的日出日落。</summary>
[Serializable]
public struct SeasonSunData
{
    public Season season;
    public float sunriseHour;        // 春秋6, 夏5, 冬7
    public float sunsetHour;         // 春秋18, 夏20, 冬17
}

/// <summary>存档规则配置。</summary>
[Serializable]
public struct SaveConfig
{
    [Tooltip("自动存档间隔（游戏内天数）")]
    public int autoSaveIntervalDays;  // 默认 3
}

/// <summary>资源包（四资源统一结构）。</summary>
[Serializable]
public struct ResourcePack
{
    public int gold;
    public int stone;
    public int wood;
    public int food;
}
```

## 四、WorldSystem 门面

**文件**：`Assets/_Game/Systems/World/WorldSystem.cs`

```csharp
using UnityEngine;

/// <summary>
/// 世界运行底层逻辑管理器（门面）。
/// 职责：持有 WorldConfig + 协调子管理器初始化 + 分发配置。
/// 不接管子管理器的运行时逻辑（TimeManager/DifficultyManager 各自管自己）。
/// </summary>
public class WorldSystem : Singleton<WorldSystem>, ISaveable
{
    public string SaveId => "WorldSystem";
    public SaveLoadPhase LoadPhase => SaveLoadPhase.Global;

    [Header("世界配置")]
    [Tooltip("Inspector 拖入，留空则从 Resources/World/ 加载")]
    [SerializeField] private WorldConfig config;

    // ===== 子管理器引用（场景里挂的 Singleton，启动后自动获取）=====

    public TimeManager Time => TimeManager.Instance;
    public DifficultyManager Difficulty => DifficultyManager.Instance;
    public WorldManager World => WorldManager.Instance;

    /// <summary>世界配置（供子管理器读取）。</summary>
    public WorldConfig Config => config;

    protected override void Awake()
    {
        base.Awake();

        // 加载配置
        if (config == null)
        {
            config = Resources.Load<WorldConfig>("World/WorldConfig");
        }
        if (config == null)
        {
            Debug.LogError("[WorldSystem] 未找到 WorldConfig！请创建资产放在 Resources/World/ 下");
        }

        SaveManager.Instance.RegisterSaveable(this);
    }

    // ===== 新建游戏：统一初始化入口 =====

    /// <summary>
    /// 新建游戏时由 GameBootstrap 调用。
    /// 统一协调：世界配置 → 难度初始化 → 时间配置 → 君主资源。
    /// </summary>
    public void InitializeWorld(NewGameConfig newConfig)
    {
        // 1. 世界管理器（地图种子 + 难度档位）
        WorldManager.Instance.ApplyConfig(newConfig.mapSeed, newConfig.difficulty);

        // 2. 难度系统初始化（设初始系数 + 同步 TimeManager 的 secondsPerDay/daysPerSeason）
        DifficultyManager.Instance.Initialize(newConfig.difficulty);

        // 3. 时间系统从 WorldConfig 读规则（秒/天已在 DifficultyManager.Initialize 里按档位设了，
        //    这里补设 daysPerSeason/dawnDuration/duskDuration/startDay/startHour）
        ApplyTimeConfig();

        // 4. 君主名字（资源由 RulerController.Awake → ApplyRulerData 读 DifficultyManager 算）
        RulerController.Instance.SetRulerName(newConfig.rulerName);

        Debug.Log($"[WorldSystem] 世界初始化完成: 难度={newConfig.difficulty}, 种子={newConfig.mapSeed}");
    }

    /// <summary>把 TimeConfig 应用到 TimeManager。</summary>
    private void ApplyTimeConfig()
    {
        var tc = config.time;
        TimeManager.Instance.SetDaysPerSeason(tc.daysPerSeason);
        // secondsPerDay 已由 DifficultyManager 按档位设（Easy 慢/Hard 快）
        // 如果想用 TimeConfig 覆盖，取消下行注释：
        // TimeManager.Instance.SetSecondsPerDay(tc.secondsPerDay);
    }

    /// <summary>把 SeasonConfig 应用到 TimeManager（日出日落表）。</summary>
    public SeasonSunData GetSeasonSunData(Season season)
    {
        return config.season.GetSeason(season);
    }

    // ===== ISaveable（WorldSystem 存档世界级状态，子管理器各自存自己的）=====

    public SavePayload SaveState()
    {
        var data = new WorldSystemSaveData
        {
            // 目前世界级状态都由子管理器各自存档，WorldSystem 暂无额外状态
            // 未来如果有跨子管理器的状态，存这里
        };
        return new SavePayload
        {
            typeName = typeof(WorldSystemSaveData).AssemblyQualifiedName,
            json = JsonUtility.ToJson(data),
            version = 1
        };
    }

    public void LoadState(SavePayload payload)
    {
        if (payload.typeName != typeof(WorldSystemSaveData).AssemblyQualifiedName) return;
        var data = JsonUtility.FromJson<WorldSystemSaveData>(payload.json);

        // 读档时也要应用配置（子管理器的 LoadState 会恢复各自状态，
        // 但 TimeManager 的 daysPerSeason 等配置项需要从 WorldConfig 重新设）
        ApplyTimeConfig();
        DifficultyManager.Instance.SyncConfigFromWorld();  // 见 DifficultyManager.LoadState

        Debug.Log("[WorldSystem] 读档恢复完成");
    }
}

[System.Serializable]
public class WorldSystemSaveData
{
    // 预留：跨子管理器的世界级状态
}
```

## 五、子管理器改造

### 5.1 TimeManager 改造

**改动**：
- 删除所有硬编码字段（secondsPerDay/startDay/startHour/daysPerSeason/DawnDuration/DuskDuration）
- 改为从 `WorldSystem.Config.time` 读取
- 删除 `totalDays`（不再是胜利条件）
- `GetSunrise/GetSunset` 改为从 `WorldSystem.GetSeasonSunData(season)` 读取
- 保留 `SeasonChangedEvent` 发布逻辑（难度系数的触发源）

```csharp
// TimeManager 关键改动
protected override void Awake()
{
    base.Awake();

    // 从 WorldConfig 读时间规则（替代硬编码）
    var tc = WorldSystem.Instance.Config.time;
    secondsPerDay = tc.secondsPerDay;
    startDay = tc.startDay;
    startHour = tc.startHour;
    daysPerSeason = tc.daysPerSeason;
    // dawnDuration / duskDuration 改为字段，从 tc 读

    CurrentDay = Mathf.Max(1, startDay);
    CurrentSeason = CalculateSeason(CurrentDay);
    // ... 其余不变 ...

    SaveManager.Instance.RegisterSaveable(this);
}

// 日出日落改为从配置读
private float GetSunrise(Season season)
{
    return WorldSystem.Instance.GetSeasonSunData(season).sunriseHour;
}

private float GetSunset(Season season)
{
    return WorldSystem.Instance.GetSeasonSunData(season).sunsetHour;
}
```

**注意**：`secondsPerDay` 会被 `DifficultyManager.Initialize` 按档位覆盖（Easy 慢/Hard 快），所以 TimeManager 先从 TimeConfig 读默认值，DifficultyManager 再按档位微调。

### 5.2 DifficultyManager 改造（v3 设计基本不变，Config 改从 WorldSystem 读）

```csharp
public class DifficultyManager : Singleton<DifficultyManager>, ISaveable
{
    public string SaveId => "DifficultyManager";
    public SaveLoadPhase LoadPhase => SaveLoadPhase.Global;

    // 不再自己持有 config，从 WorldSystem 读
    private DifficultyConfig Config => WorldSystem.Instance.Config.difficulty;

    public int CurrentDifficulty { get; private set; }
    public float CurrentFactor { get; private set; }

    protected override void Awake()
    {
        base.Awake();
        SaveManager.Instance.RegisterSaveable(this);
    }

    private void Start()
    {
        EventBus.Subscribe<SeasonChangedEvent>(OnSeasonChanged);
    }

    private void OnDestroy()
    {
        EventBus.Unsubscribe<SeasonChangedEvent>(OnSeasonChanged);
    }

    /// <summary>新建游戏初始化。</summary>
    public void Initialize(int difficulty)
    {
        CurrentDifficulty = Mathf.Clamp(difficulty, 1, 3);
        CurrentFactor = CurrentDifficulty;

        // 按档位同步 TimeManager 的 secondsPerDay
        DifficultyPreset preset = Config.GetPreset(CurrentDifficulty);
        TimeManager.Instance.SetSecondsPerDay(preset.secondsPerDay);

        Debug.Log($"[DifficultyManager] 初始化: 档位={CurrentDifficulty}, 系数={CurrentFactor}");
    }

    /// <summary>读档后同步配置（由 WorldSystem.LoadState 调用）。</summary>
    public void SyncConfigFromWorld()
    {
        DifficultyPreset preset = Config.GetPreset(CurrentDifficulty);
        TimeManager.Instance.SetSecondsPerDay(preset.secondsPerDay);
    }

    public ResourcePack GetInitialResources()
    {
        return Config.GetInitialResources(CurrentDifficulty);
    }

    private void OnSeasonChanged(SeasonChangedEvent evt)
    {
        float oldFactor = CurrentFactor;
        CurrentFactor += Config.factorGrowthPerSeason;
        EventBus.Publish(new DifficultyChangedEvent(oldFactor, CurrentFactor));
    }

    // ISaveable（不变）
    public SavePayload SaveState() { /* 同 v3 */ }
    public void LoadState(SavePayload payload) { /* 同 v3，恢复后调 SyncConfigFromWorld */ }
}
```

## 六、和其他系统的结合

### 6.1 存档系统（SaveManager）

**不改动**。各子管理器各自 `ISaveable`：
- `WorldSystem` → `WorldSystemSaveData`（预留，目前空）
- `TimeManager` → `TimeSaveData`（删 totalDays 字段）
- `DifficultyManager` → `DifficultySaveData`（currentDifficulty + currentFactor）
- `WorldManager` → `WorldSaveData`（删 totalDays）
- `RulerController` → `RulerSaveData`（不变）

`SaveManager` 自动收集，**不需要改代码**。

### 6.2 GameBootstrap 改造

**改动**：
- `InitializeScene()` 里删除 `TimeManager.SetTotalDays` 和 `WorldManager.ApplyConfig(..., totalDays)`
- 新增 `WorldSystem.Instance.InitializeWorld(NewGameConfig)` 统一调用
- 初始化顺序：WorldSystem.Awake（最早）→ 各子管理器.Awake → WorldSystem.InitializeWorld

```csharp
// GameBootstrap.InitializeScene 改动后
private void InitializeScene()
{
    if (IsLoadingSave)
    {
        SaveManager.Instance.Load(SaveSlotToLoad);
        // 各子管理器通过 ISaveable 自动恢复
        // WorldSystem.LoadState 会调 ApplyTimeConfig + DifficultyManager.SyncConfigFromWorld
    }
    else
    {
        if (NewGameConfig != null)
        {
            // 统一入口：WorldSystem 协调所有子管理器初始化
            WorldSystem.Instance.InitializeWorld(NewGameConfig);
        }
        RulerController.Instance.SpawnMonarch();
    }

    InputManager.Instance.EnableInput();
    GameStateManager.Instance.SetState(GameState.Playing);

    // 新建游戏后自动存档
    if (!IsLoadingSave)
    {
        string slotId = NewGameConfig?.selectedSlotId ?? "slot_1";
        if (!SaveManager.Instance.HasSave(slotId))
        {
            SaveManager.Instance.Save(slotId);
        }
        NewGameConfig = null;
    }
}
```

| CombatSystem | 伤害计算参考 `CurrentFactor` | 战斗系统开发时 |
| EnemyController | 属性初始化 × `CurrentFactor` | 敌人波次开发时 |
| SaveManager | 自动存档间隔读 `WorldSystem.Config.save.autoSaveIntervalDays` | 存档系统调整时 |

## 七、数据流

### 7.1 新建游戏

```
CharacterCreationPanel 选难度(1/2/3)
  │
  ▼
NewGameConfig { rulerName, difficulty, mapSeed }
  │
  ▼
GameBootstrap.InitializeScene()
  │
  └─ WorldSystem.InitializeWorld(NewGameConfig)
       ├─ WorldManager.ApplyConfig(mapSeed, difficulty)
       ├─ DifficultyManager.Initialize(difficulty)
       │    ├─ CurrentFactor = difficulty（1/2/3）
       │    └─ TimeManager.SetSecondsPerDay(按档位)
       ├─ ApplyTimeConfig()（daysPerSeason 等从 WorldConfig 读）
       └─ RulerController.SetRulerName(name)
            └─ RulerController.Awake → ApplyRulerData
                 └─ 初始资源 = WorldConfig.difficulty.GetInitialResources(difficulty)
```

### 7.2 读档

```
SaveManager.Load(slotId)
  │
  ├─ Global 阶段：
  │    ├─ WorldSystem.LoadState → ApplyTimeConfig + DifficultyManager.SyncConfigFromWorld
  │    ├─ TimeManager.LoadState → 恢复 CurrentDay/Season/时间配置
  │    ├─ DifficultyManager.LoadState → 恢复 CurrentDifficulty/CurrentFactor
  │    └─ WorldManager.LoadState → 恢复 MapSeed/Difficulty
  │
  └─ Scene 阶段：
       └─ RulerController.LoadState → 恢复资源数值
```

### 7.3 运行时

```
TimeManager 每过 daysPerSeason 天
  │
  ▼
SeasonChangedEvent
  │
  ▼
DifficultyManager.OnSeasonChanged()
  ├─ CurrentFactor += 0.25
  └─ DifficultyChangedEvent
       ├─→ WaveManager（未来）：敌人强度 × newFactor
       └─→ UI（可选）：显示难度系数
```
