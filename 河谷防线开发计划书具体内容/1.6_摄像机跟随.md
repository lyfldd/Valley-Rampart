# 1.6 摄像机跟随

### 优先级：🟡 中（依赖 1.5 君主移动）

### 脚本

| 文件 | 路径 | 说明 |
|------|------|------|
| `CameraFollow.cs` | `Assets/_Game/Systems/Camera/` | 摄像机平滑跟随 + 视差滚动驱动 |
| `ParallaxLayer.cs` | `Assets/_Game/Systems/Camera/` | 视差层组件，挂载到需要视差的背景 Tilemap 上 |

### CameraFollow 核心逻辑

```
CameraFollow : MonoBehaviour
  挂载：Main Camera 上

  字段：
    Transform target          ← 拖入 Ruler
    float smoothSpeed = 5f
    Vector3 offset            ← 初始时自动计算
    float minX, maxX          ← 地图边界（硬编码，后期自动计算）
    Vector3 lastFramePos      ← 上一帧摄像机位置（用于计算视差偏移量）

  LateUpdate() {
    1. 计算目标位置 + Clamp X 轴
    2. Lerp 平滑跟随
    3. 广播 deltaX = transform.position.x - lastFramePos.x
    4. 更新 lastFramePos
  }
```

### ParallaxLayer 核心逻辑

```
ParallaxLayer : MonoBehaviour
  挂载：SkyBg / TerrainBg 等需要视差的 Tilemap GameObject 上

  字段：
    public float parallaxFactor = 1f
        0 = 完全静止（天空）
        0.2 = 缓慢移动（远山背景）
        1 = 跟随摄像机（地表/剖面/建筑，与地面同步）

  // 在 LateUpdate 中由 CameraFollow 调用
  public void ApplyParallax(float cameraDeltaX) {
      transform.position += Vector3.right * cameraDeltaX * (1f - parallaxFactor)
  }
```

### 各层 parallaxFactor 预设

| 层 | Factor | 效果 |
|----|:------:|------|
| SkyBg | 0 | 天空完全不动 |
| TerrainBg | 0.2 | 远山缓慢跟随，产生纵深感 |
| Surface | 1 | 跟摄像机完全同步 |
| TerrainProp | 1 | 跟地表同步 |
| Water | 1 | 跟地表同步 |
| Decoration | 1 | 游戏对象跟地表同步 |

### 实现要点

1. 挂载在主摄像机 `Main Camera` 上
2. `target` 在 Inspector 中拖入 Ruler
3. `minX` / `maxX` 根据 Tilemap 实际宽度手动设置（现阶段硬编码，后期改为自动计算）
4. Y 轴锁定，横版游戏不需要上下跟随
5. 用 `LateUpdate` 而非 `Update`——确保在君主移动完成后计算
6. 视差层：SkyBg 和 TerrainBg 的 Tilemap GameObject 挂 `ParallaxLayer`，设置对应 factor
7. CameraFollow 的 `LateUpdate` 末尾计算 deltaX，遍历所有 ParallaxLayer 调用 ApplyParallax
8. 视差偏移只改 transform.position.x，无需额外相机或渲染设置

### 验证标准
- [ ] 君主移动时摄像机平滑跟随
- [ ] 君主到达地图边界时摄像机停止移动（不超出 Tilemap 范围）
- [ ] Y 轴不晃动
- [ ] 天空层完全静止不随摄像机移动
- [ ] 远山背景以明显慢于地表的速度移动，产生纵深感
- [ ] 地表/剖面/建筑跟摄像机完全同步，无拖影
