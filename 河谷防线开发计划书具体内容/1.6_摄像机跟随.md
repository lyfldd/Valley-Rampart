# 1.6 摄像机跟随

### 优先级：🟡 中（依赖 1.5 君主移动）

### 脚本

| 文件 | 路径 | 说明 |
|------|------|------|
| `CameraFollow.cs` | `Assets/_Game/Systems/Camera/` | 摄像机平滑跟随 |

### 核心逻辑

```
CameraFollow : MonoBehaviour
  挂载：Main Camera 上

  字段：
    Transform target          ← 拖入 Ruler
    float smoothSpeed = 5f
    Vector3 offset            ← 初始时自动计算
    float minX, maxX          ← 地图边界（硬编码，后期自动计算）

  LateUpdate() {
    Vector3 desiredPos = target.position + offset
    desiredPos.x = Mathf.Clamp(desiredPos.x, minX, maxX)
    desiredPos.y = transform.position.y（Y 轴不跟随）
    transform.position = Vector3.Lerp(transform.position, desiredPos, smoothSpeed * Time.deltaTime)
  }
```

### 实现要点

1. 挂载在主摄像机 `Main Camera` 上
2. `target` 在 Inspector 中拖入 Ruler
3. `minX` / `maxX` 根据 Tilemap 实际宽度手动设置（现阶段硬编码，后期改为自动计算）
4. Y 轴锁定，横版游戏不需要上下跟随
5. 用 `LateUpdate` 而非 `Update`——确保在君主移动完成后计算

### 验证标准
- [ ] 君主移动时摄像机平滑跟随
- [ ] 君主到达地图边界时摄像机停止移动（不超出 Tilemap 范围）
- [ ] Y 轴不晃动
