## v2 核心决策（阿铁拍板）

1. **图层简化**：背景统一大图 + 剖面（纯视觉前景）+ 游戏内容层。砍掉 v1 的"图层1背景"独立层。
2. **剖面性质**：纯视觉前景，不参与玩法。**地形改造系统整体删除**（挖壕沟/注水/造隘口/埋地雷全砍）。
3. **地图生成**：三层约束程序化生成（大地形→小地形→资源点），保留 mapSeed，存档存 seed 复现。
4. **资源产出**：伐木场（依赖树资源点）/ 矿洞改造（依赖地图矿洞）/ 农场（无地形依赖）三类产出建筑。
5. **地形类型**：玩家开局选岛屿（两边出怪）或内陆（一边出怪，一边大山）；敌方王国随机生成；无限关卡固定岛屿。
6. **通关流程**：初始王国 → 昼夜循环 → 造船远征 → 攻打随机敌方王国 → 占领跨岛移动 → 全通关 → 无限塔防。

## 一、图层结构

```
┌─────────────────────────────────────────────┐
│  游戏内容层（Layer 2）                        │
│  玩家 / 建筑 / 敌人 / 资源点 sprite / UI      │
│  ← 所有可交互对象在这层                       │
├─────────────────────────────────────────────┤
│  剖面前景层（Layer 1）                        │
│  土壤剖面视觉（纯装饰，不参与玩法）            │
│  矿洞开口画在这层的基准线上                    │
├─────────────────────────────────────────────┤
│  背景大图层（Layer 0）                        │
│  统一背景大图（按小地形类型区分：林地全树/    │
│  荒地稀疏/矿山岩壁等）                        │
└─────────────────────────────────────────────┘
```

### 各层职责

| 层级 | 内容 | 是否参与玩法 |
|------|------|:--:|
| Layer 0 背景大图 | 按小地形类型的手绘背景（林地→全树 / 荒地→稀疏 / 矿山→岩壁）。树资源点的视觉表现层 | 间接（树资源点的视觉锚点） |
| Layer 1 剖面前景 | 土壤剖面视觉 + 矿洞开口 sprite | 否（纯视觉），矿洞开口是 Layer 2 矿洞建筑的视觉锚点 |
| Layer 2 游戏内容 | 玩家/建筑/敌人/资源点/UI | 是 |

### 剖面基准线

- 基准线 y 值由**开发者自定义**（不锁定具体值，阿铁开发时自行调整）
- 基准线以上：游戏内容层（玩家/建筑/敌人活动区）
- 基准线以下：剖面视觉（土壤层次，纯装饰）
- 矿洞开口画在基准线上，玩家在其上建/改造矿洞建筑

## 二、地图生成：三层约束程序化生成

```
第1层 大地形类型（玩家开局选 / 关卡决定）
  ├─ 岛屿 → 两边出怪（模拟敌军登陆）
  └─ 内陆 → 一边出怪（另一边是大山，天然屏障）
        │
        ▼ 约束
第2层 小地形（程序化拼接，受大地形约束）
  ├─ 平原 / 肥沃 / 荒地 / 丘陵 / 林地 / 矿山 / 雪山
  └─ 岛屿倾向沿海地形（平原/肥沃多），内陆倾向多样地形
        │
        ▼ 约束
第3层 资源点（程序化放置，受小地形约束）
  ├─ 树资源点（林地/平原）→ 伐木场依赖（不占建造位，范围判定）
  ├─ 矿洞开口（矿山/丘陵基准线）→ 矿洞建筑依赖
  ├─ 农田位（肥沃地形）→ 农场依赖
  └─ 未来扩展：遗迹 / 宝箱 / 特殊地标
```

### 生成算法要求

- **确定性**：同 seed 同结果（用 `System.Random(mapSeed)`，禁用 `UnityEngine.Random`）
- **WorldManager.GenerateWorld(mapSeed)** 落地此算法（当前是 TODO）
- 存档存 mapSeed，读档用同 seed 复现地形
- 三层约束的"约束规则"用 ScriptableObject 配置（可调，不改代码）

### 地形类型约束表【待 design-strategist 细化】

| 大地形 | 可出现的小地形 | 资源点倾向 |
|--------|--------------|-----------|
| 岛屿 | 平原/肥沃/丘陵/林地（少矿山雪山） | 树木多、矿洞少 |
| 内陆 | 全部 7 种 | 树木/矿洞/农田均匀 |
| 无限关卡（固定岛屿） | 平原/丘陵（开阔地利于塔防） | 资源点稀疏（压力大） |

## 三、资源产出机制

### 三类产出建筑

| 建筑 | 产出 | 地形依赖 | 机制 |
|------|------|---------|------|
| **伐木场** | 木头 | 树资源点的范围判定内 | 树是基准线上的资源点，**不占建造位**，伐木场建在其范围判定内即持续产木；视觉用特殊背景表现 |
| **矿洞** | 石头/矿石 | 基准线有矿洞开口（矿山/丘陵） | 玩家改造矿洞（升级），持续产石；矿洞是地图自带资源点 |
| **农场** | 粮食 | 肥沃地形 | 纯建造物，无地图资源点依赖 |
| 金币 | — | — | 待定（税收/贸易/占领产出） |

### 关键设计：树资源点

- **树 = 基准线上的资源点**，但**不占建造位**——伐木场建造时不占用树的位置，二者共存
- **范围判定**：通过判定半径（伐木场的采集半径 or 树的供养半径）决定是否产出，逻辑上灵活可调
- **视觉表现**：树用**特殊背景**表现，不是独立 sprite。解决阿铁提出的"放剖面上表现形式不好"问题
- **矿洞是资源点**：地图生成时在矿山/丘陵地形的基准线放矿洞开口 sprite，玩家点击矿洞→改造（升级）→持续产石
- **农田无依赖**：农场建在肥沃地形上即可，不依赖地图资源点

## 四、地形类型与出怪方向

### 玩家开局选择

| 地形 | 出怪方向 | 难度 | 说明 |
|------|---------|:--:|------|
| 岛屿 | 两边 | 高 | 模拟敌军登陆，需双向防御 |
| 内陆 | 一边（另一边大山） | 低 | 大山是天然屏障，单向防御 |

### 敌方王国

- **随机生成**：需要攻击的敌方王国按 mapSeed 程序化生成（地形/兵力/资源）
- **数量**：地图生成的**全部**敌方王国（数量由生成参数决定，非固定 3-5 个），全部击败算通关

### 无限关卡

- **固定岛屿地形**：两边出怪，难度最高
- **通关后进入**：全通关后存档保留，进入无限塔防模式
- **难度递增**：按难度系统公式无限增长

## 五、通关流程

```
开局
  │  玩家选岛屿/内陆 + 少量初始资源 + 简易王国
  ▼
昼夜循环（无数天）
  │  白天：建造/采集/经营
  │  晚上：出怪（岛屿两边/内陆一边）
  │  难度线性增长：初始系数3，每季+0.25，每年（4季）+1.0
  ▼
造船 + 远征
  │  建造船厂 → 造船 → 船可在当前地图使用 or 选择前往下一岛屿
  │  跨岛移动：君主亲自带队 → 转场动画 → 时间跳到下一天清晨
  ▼
攻打敌方王国
  │  抵达目标岛屿/内陆 → 攻城 → 占领
  │  占领后可在己方领土间自由移动
  ▼
全通关
  │  击败全部敌方王国 → 通关
  │  存档保留
  ▼
无限塔防模式
  │  固定岛屿地形，两边出怪
  │  难度按公式无限递增，直到玩家阵亡
```

### 难度系统

- **线性增长**，绑定季节变化（游戏已有季节系统 = 已有"年"的单位）
- **初始难度系数** = 3
- **每季** +0.25（一季约若干天，由 TimeManager 配置）
- **每年**（4季）+1.0
- 难度系数影响：敌人血量/数量/类型/刷新频率（具体映射表待 design-strategist 细化）
- 无限关卡模式下，难度系数继续按此公式无限增长

### 造船厂建筑

- **建筑**：造船厂（玩家建造，非地图固定位置）
- **造船**：造船厂建好后，可生成船单位
- **船的用途**：
  - 当前地图游戏内容（可在当前地图上使用）
  - 选择前往下一个岛屿（跨岛远征的载体）
- 没造船厂 = 没船 = 无法远征

### 跨岛移动机制

- **君主亲自参与**：跨岛需要君主带队（非纯菜单操作）
- **转场动画**：触发过场动画表现航行
- **时间跳跃**：动画结束后，时间直接跳到当前记录天数的**下一天清晨**
- 非实时航行，非菜单瞬时到达——介于两者之间：有过程感但不拖沓

## 七、对开发计划书的影响

### 删除的系统

- ❌ **地形改造系统**（挖壕沟/注水/造隘口/埋地雷）—— v2 整体删除，剖面降为纯视觉

### 修改的系统

- 🔄 **Tilemap 河谷地图** → **地图生成系统**：从"7张复合地形大图横向拼接"改为"三层约束程序化生成"
- 🔄 **WorldManager**：`GenerateWorld(mapSeed)` 从 TODO 落地为三层约束算法
- 🔄 **建造系统**：新增"资源产出建筑"概念（伐木场/矿洞/农场/造船厂），区分地形依赖
- 🔄 **跨岛远征**：细化为"造船厂→造船→君主带队转场→时间跳到下一天清晨→攻城→占领"
- 🔄 **资源经济**：木/石/粮产出分别绑定伐木场/矿洞/农场，金币产出待定
- 🔄 **难度系统**：新增线性难度系数（初始3，每季+0.25，每年+1.0），绑定季节

### 新增的概念

- ✅ 地形类型选择（岛屿/内陆）
- ✅ 无限塔防模式（通关后）
- ✅ 造船厂建筑 + 船单位（当前地图使用 / 跨岛远征载体）
- ✅ 树资源点（基准线上，不占建造位，范围判定，特殊背景视觉）
- ✅ 敌方王国程序化生成（全部击败算通关）
- ✅ 跨岛转场动画 + 时间跳跃机制
