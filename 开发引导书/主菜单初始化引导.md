Unity 启动后加载 MainMenuScene
最先被引导的就是CoreBootstrap，这个是引擎层引导器
负责创建 Core 层单例：GameStateManager、EventBus、InputManager、SaveManager 等不做任何业务初始化（不加载 UnitDataManager、不创建单位、不挂业务系统）

接下来是单例管理器的
由 CoreBootstrap 主动创建的
GameStateManager，InputManager，SaveManager，WorldManager
创建顺序严格按照 CoreBootstrap 中代码的书写顺序
var _   = GameStateManager.Instance;   // 第1个
var __  = InputManager.Instance;       // 第2个
var ___ = SaveManager.Instance;        // 第3个
var ____ = WorldManager.Instance;      // 第4个

由其他代码间接触发的
RulerController
加载君主 RulerData，注册 ISaveable，订阅 UnitDiedEvent
TimeManager
初始化天数/季节/时段，注册 ISaveable
UnitDataManage
从 Resources 加载所有单位配置数据
UnitFactory
预加载单位 Prefab
UnitRegistry
维护场景中所有单位的查找表

创建时间线对比
CoreBootstrap.Awake() [-200]
  ├─ 主动创建 ▼
  ├── GameStateManager    ← 当场 AddComponent → Awake
  ├── InputManager        ← 当场 AddComponent → Awake
  ├── SaveManager         ← 当场 AddComponent → Awake
  └── WorldManager        ← 当场 AddComponent → Awake
        │
        ▼ (CoreBootstrap 结束)
        
GameBootstrap.Awake() [-100]
  └─ UnitFactory.Instance ← 首次访问，当场 AddComponent → Awake

GameBootstrap.Start()
  ├─ UnitDataManager.Instance ← 首次访问，当场 AddComponent → Awake
  └─ GameStateManager.SetState(Loading)
        │
        ▼
        
GameBootstrap.InitializeScene()
  ├─ TimeManager.Instance     ← 首次访问（如果场景没挂）
  └─ RulerController.Instance ← SpawnMonarch 中首次引用
