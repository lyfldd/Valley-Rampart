# 主菜单 / 场景初始化引导

> 适用：MainMenuScene 与 GameScene 的启动链路。
> 两个场景都挂 `CoreBootstrap`；GameScene 额外挂 `GameBootstrap`；MainMenuScene 挂 `MainMenuController`。
> ⚠️ 旧注释里说的 "MainMenuScene 挂 MainMenuBootstrap" 是错的——那个类根本不存在，主菜单靠 `MainMenuController` 推进。
> 加载协调由 `LoadManager`（-110）统一接管，业务引导器只负责触发与分叉。

## 1. 总原则

- `CoreBootstrap`（引擎层，`DefaultExecutionOrder(-200)`）只创建 Core 层单例，**不做任何业务初始化**（不加载 UnitDataManager、不创建单位、不挂业务系统）。
- `EventBus` 是 **`static class`，不是 Singleton**，**不需要也不由 CoreBootstrap 创建**，进程启动即可用。旧版把它写进 CoreBootstrap 创建列表是错的。
- `LoadManager`（`DefaultExecutionOrder(-110)`）是**加载总管**：阶段化调度静态配置加载、新建/读档恢复，并统一 `SetState(Playing)`。各子加载器（UnitDataManager / UnitFactory / SaveManager）和子管理器（WorldSystem）只干活，不自己决定时机。
  - `UnitDataManager` / `UnitFactory` **不再在 Awake 自动加载**，改由 `LoadManager.LoadStaticConfigs()` 显式调。
  - 加载完成事件已由 `UnitDataLoadedEvent` 改为 `ConfigsLoadedEvent`（由 `LoadManager` 发布）。
- 业务层（`MainMenuController` / `GameBootstrap`）在 Core 就绪后各自推进状态机。

## 2. CoreBootstrap.Awake() [-200]：主动创建的 4 个单例

顺序严格按代码书写（Singleton<T>.Instance 首次访问时内部 `new GameObject + AddComponent` → 立即走 Awake，DontDestroyOnLoad）：

```csharp
var _    = GameStateManager.Instance;  // 1
var __   = InputManager.Instance;      // 2
var ___  = SaveManager.Instance;       // 3
var ____ = WorldManager.Instance;      // 4  注意：是世界地图管理器，不是 WorldSystem
```

- 只有**首次**启动执行 `SetState(Booting)`（由静态守卫 `_isFirstCoreBootstrap` 控制）。跨场景切回时 `GameStateManager` 已是 DontDestroyOnLoad 单例，状态从上个场景延续，不再覆盖。

## 3. MainMenuScene 初始化链路（主菜单）

1. `CoreBootstrap.Awake [-200]` → Core 单例就绪，首次启动 `SetState(Booting)`。
2. `MainMenuController.Start` → `SetState(Splash)`，显示过场面板。
3. Splash 结束 `OnSplashCompleted()` → `ShowPanel(MainMenu)` → `SetState(MainMenu)`。
4. 玩家选择：
   - 新游戏 → `CharacterCreation` 面板 → `OnCharacterCreateConfirmed(config)` → `GameSceneEntrance.SetNewGame(config)`。
   - 继续游戏 → `SaveSlots` 面板 → `OnSaveSlotSelected(slotId)` → `GameSceneEntrance.SetContinue(slotId)`。
5. `LoadGameScene()` → `InputManager.DisableInput()` → `SceneManager.LoadScene("GameScene")`。

> 主菜单状态机：`Booting → Splash → MainMenu → (CharacterCreation | SaveSlotSelect)`。

## 4. GameScene 初始化链路（LoadManager 协调）

### 4.1 GameBootstrap.Awake() [-100]

- `UnitRegistry.Instance.Clear()`：清掉上一场景残留引用（跨场景清理 R6）。
- `SaveManager.Instance.CleanupDestroyedSaveables()`：清理已销毁的 ISaveable。
- `SaveManager.Instance.RegisterSpawner(UnitFactory.Instance)`：注册场景对象重建器（**这里首次触碰 UnitFactory，触发其 Awake**）。
- `EventBus.Subscribe<ConfigsLoadedEvent>(OnConfigsLoaded)`：订阅静态配置加载完成事件（替代旧的 `UnitDataLoadedEvent`）。

> 注意执行顺序：`LoadManager` 是 -110，比 `GameBootstrap`(-100) 更早 Awake。LoadManager.Awake 只做引用获取（要求 UnitDataManager/UnitFactory/WorldSystem 在场景中显式放置），不触发加载。

### 4.2 LoadManager 阶段调度（加载总管）

| 阶段 | 方法 | 做了什么 |
|------|------|----------|
| 阶段1 静态配置 | `LoadStaticConfigs()` | `UnitDataManager.LoadAll()`（Resources 加载全部单位 SO）+ `UnitFactory.PreloadAll()`（预加载 Prefab）→ 发布 `ConfigsLoadedEvent` |
| 阶段2a 新建 | `InitializeNewGame(config)` | `WorldSystem.InitializeWorld(config)`（协调 WorldManager+DifficultyManager+TimeManager+君主资源）→ `RulerController.SpawnMonarch()` → `EnterPlaying()` |
| 阶段2b 读档 | `LoadSave(slotId)` | `SaveManager.Load(slotId)`（各 ISaveable 自行恢复 + WorldSystem 应用配置）→ `EnterPlaying()` |
| 阶段3 运行时 | `EnterPlaying()` | `SetState(Playing)`，此后可按需动态加载 |

`WorldSystem.InitializeWorld` 内部顺序：
1. `WorldManager.ApplyConfig(seed, difficulty)`（地图种子 + 难度档）
2. `DifficultyManager.Initialize(difficulty)`（初始系数 + 设 TimeManager.secondsPerDay）
3. `WorldSystem.ApplyTimeConfig()`（daysPerSeason）
4. `RulerController.SetRulerName` + `ApplyInitialResourcesFromDifficulty()`

### 4.3 GameBootstrap.Start()

- `SetState(Loading)`。
- `LoadManager.Instance.LoadStaticConfigs()`：触发阶段1（同步加载配置 + Prefab，完成后发 `ConfigsLoadedEvent`）。

### 4.4 OnConfigsLoaded → 分叉

收到 `ConfigsLoadedEvent` 后 `SetState(Ready)`，再按 `GameSceneEntrance.CurrentMode` 分叉：

- **NewGame** → `StartNewGame()`：
  - `LoadManager.Instance.InitializeNewGame(config)`：阶段2a（世界初始化 + 君主生成 + `SetState(Playing)`）。
  - 若该槽无存档则 `SaveManager.Save(slotId)` 自动存初始档。
- **ContinueGame** → `ContinueFromSave()`：
  1. `RulerController.DestroyAllSceneUnits()` 清掉场景预置单位。
  2. `LoadManager.Instance.LoadSave(slotId)`：阶段2b（读档恢复 + `SetState(Playing)`）。
  3. `RulerController.BindExistingMonarch()` 绑定恢复出的君主。

分叉结束后：
- `TimeManager.Instance` 时间系统就绪（日志）。
- `InputManager.Instance.EnableInput()`。
- `GameSceneEntrance.Clear()` 清理跨场景静态参数。

> 游戏内状态机：`Loading → Ready → Playing → (Paused | GameOver)`（`Playing` 由 LoadManager.EnterPlaying 设置）。

## 5. 单例触发来源总表

| 单例 | 触发方式 | 触发点 |
|------|----------|--------|
| GameStateManager | CoreBootstrap 主动创建 | CoreBootstrap.Awake #1 |
| InputManager | CoreBootstrap 主动创建 | CoreBootstrap.Awake #2 |
| SaveManager | CoreBootstrap 主动创建 | CoreBootstrap.Awake #3 |
| WorldManager | CoreBootstrap 主动创建 | CoreBootstrap.Awake #4 |
| UnitFactory | GameBootstrap 主动触碰（RegisterSpawner）；实际预加载由 LoadManager 调 | GameBootstrap.Awake / LoadManager 阶段1 |
| UnitDataManager | 不再 Awake 自加载；由 `LoadManager.LoadStaticConfigs` 调 `LoadAll` | LoadManager 阶段1 |
| UnitRegistry | GameBootstrap 主动 Clear | GameBootstrap.Awake |
| LoadManager | 场景显式放置；GameBootstrap.Start 访问 | GameBootstrap.Start |
| WorldSystem | LoadManager.Awake 经 FindObjectOfType 获取；InitializeNewGame 调 | LoadManager / 阶段2a |
| DifficultyManager | WorldSystem.InitializeWorld 内部访问 | 阶段2a |
| TimeManager | WorldSystem 协调 / OnConfigsLoaded 末访问 | 阶段2a / OnConfigsLoaded |
| RulerController | SpawnMonarch / BindExistingMonarch（内部经 LoadManager 取数据、生成单位） | 阶段2a / 阶段2b |

## 6. 执行顺序速查

```
CoreBootstrap.Awake        [-200]  创建 4 单例 + 首次 SetState(Booting)
  └─ 两场景都跑；跨场景只首次设 Booting

MainMenuScene:
  MainMenuController.Start         SetState(Splash → MainMenu → …)

GameScene:
  LoadManager.Awake         [-110]  获取子加载器引用（UnitDataManager/UnitFactory/WorldSystem）
  GameBootstrap.Awake       [-100]  UnitRegistry.Clear / Cleanup / RegisterSpawner(UnitFactory) / Subscribe(ConfigsLoadedEvent)
  WorldSystem.Awake         [-95]   Resources.Load(WorldConfig) + RegisterSaveable
  GameBootstrap.Start               SetState(Loading) → LoadManager.LoadStaticConfigs()
  LoadManager 阶段1                 UnitDataManager.LoadAll + UnitFactory.PreloadAll → ConfigsLoadedEvent
  GameBootstrap.OnConfigsLoaded     SetState(Ready) → 按模式分叉
    ├─ NewGame    → LoadManager.InitializeNewGame  → SetState(Playing) + 自动存初始档
    └─ Continue   → DestroyAllSceneUnits → LoadManager.LoadSave → BindExistingMonarch
  OnConfigsLoaded 收尾              TimeManager 就绪 / InputManager.EnableInput / GameSceneEntrance.Clear
```
