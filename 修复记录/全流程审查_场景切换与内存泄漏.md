## 0. 检查方法

对每个 MonoBehaviour 检查四条配对：
1. **EventBus 订阅 ↔ 退订**：Awake/OnEnable 订阅 vs OnDestroy/OnDisable 退订
2. **UI Toolkit clicked += ↔ -=**：OnEnable 绑定 vs OnDisable 解绑
3. **创建 ↔ 销毁**：Instantiate/Spawn vs Destroy/Clear
4. **注册 ↔ 注销**：RegisterSaveable/Register vs UnregisterSaveable/Unregister/Clear

## 1. 发现的问题

### P0-1 · 跨场景清理逻辑丢失（R6 回归）⚠️ 最严重

**位置**：`GameBootstrap.Awake`

**现象**：之前的 GameBootstrap.Awake 有这两行：
```csharp
UnitRegistry.Instance.Clear();
SaveManager.Instance.CleanupDestroyedSaveables();
```
重构为 GameSceneEntrance 桥接模式后，这两行**被删除了**。`SaveManager.CleanupDestroyedSaveables()` 方法和 `UnitRegistry.Clear()` 方法都还在，但 **没有任何地方调用它们**（已用 Grep 确认）。

**后果**：玩家从 GameScene 退回主菜单 → 再进 GameScene 时：
1. 旧场景的单位被 Unity 销毁，但 `_saveables` 字典仍持有 C# 引用 → **内存泄漏**
2. `_aliveUnits` HashSet 持有已销毁单位 → **幽灵引用**
3. `UnitRegistry.GetEnemies()` / `GetNearestEnemy()` 返回已销毁对象 → 调用方访问 `unit.transform.position` 报 **MissingReferenceException**
4. `SaveManager.Save` 有 `if (mb == null) continue` 防御不会崩，但存档时遍历的条目越来越多

**触发路径**：
```
GameScene (Playing) → PausePanel.OnQuitClicked → LoadScene(MainMenuScene)
  → 旧场景单位被 Unity 销毁，_saveables / _aliveUnits 不清理
  → MainMenuScene → OnSaveSlotSelected → LoadScene(GameScene)
  → GameBootstrap.Awake 不清理 → 旧引用残留 → 新场景里 GetEnemies 返回幽灵
```

**修复**：在 `GameBootstrap.Awake` 顶部加回清理：
```csharp
private void Awake()
{
    Debug.Log("[GameBootstrap] 游戏场景启动");

    // R6: 跨场景清理——清除上一场景残留的已销毁引用
    UnitRegistry.Instance.Clear();
    SaveManager.Instance.CleanupDestroyedSaveables();

    SaveManager.Instance.RegisterSpawner(UnitFactory.Instance);
    EventBus.Subscribe<UnitDataLoadedEvent>(OnDataLoaded);
}
```

### P1-1 · UI Toolkit 按钮 clicked += 重复注册（4 个面板）

**位置**：`PausePanel.OnEnable`、`GameOverPanel.OnEnable`、`SaveSlotPanel.OnEnable`、`CharacterCreationPanel.OnEnable`

**现象**：`OnEnable` 每次都执行 `button.clicked += handler`，既没有 `_bound` 检查，也没有 `OnDisable` 里 `clicked -=` 退订。

`MainMenuController.ShowPanel` 用 `SetActive` 切换面板，每次切换都会触发 `OnEnable`：
```csharp
// MainMenuController.ShowPanel
if (saveSlotsRoot != null) saveSlotsRoot.SetActive(panel == Panel.SaveSlots);
// → SaveSlotPanel.OnEnable 被调用 → slots-back-button.clicked += 又注册一次
```

**后果**：进出面板 N 次 → 按钮回调注册 N 次 → 点一次按钮触发 N 次回调。

**各面板具体情况**：

| 面板 | 绑定的按钮 | 有 _bound 检查？ | 有 OnDisable 退订？ |
|------|-----------|-----------------|-------------------|
| PausePanel | resume / save / quit | ❌ OnEnable 无检查，Start 有 | ❌ |
| GameOverPanel | back-to-menu | ❌ OnEnable 无检查，Start 有 | ❌ |
| SaveSlotPanel | slots-back | ❌ | ❌ 完全没 OnDisable |
| CharacterCreationPanel | creation-back / creation-confirm | ❌ | ❌ 完全没 OnDisable |

**修复方案**：每个面板加 `OnDisable` 退订，并在 `OnEnable` 里绑定前先检查：

```csharp
private bool _buttonsBound;

private void OnEnable()
{
    EventBus.Subscribe<...>(...);
    if (!_buttonsBound) BindButtons();
    SetPanelVisible(false);
}

private void OnDisable()
{
    EventBus.Unsubscribe<...>(...);
    // UI Toolkit 的 clicked 是 event，需要 -= 退订
    if (_buttonsBound)
    {
        var root = GetComponent<UIDocument>().rootVisualElement;
        root.Q<Button>("xxx").clicked -= OnXxx;
        _buttonsBound = false;
    }
}
```

### P1-2 · CharacterCreationPanel 的 RegisterValueChangedCallback 没有退订

**位置**：`CharacterCreationPanel.OnEnable`

**现象**：
```csharp
_totalDaysSlider.RegisterValueChangedCallback(evt => UpdateTotalDaysLabel(evt.newValue));
_difficultyGroup.RegisterValueChangedCallback(OnDifficultyChanged);
_slotGroup.RegisterValueChangedCallback(_ => RefreshSlotAvailability());
```
三处 `RegisterValueChangedCallback` 都没有对应的 `UnregisterValueChangedCallback`。

**后果**：同 P1-1，反复 SetActive 导致值变化回调注册多次，滑块拖动时触发多次刷新。

**修复**：保存回调引用，在 OnDisable 里 `UnregisterValueChangedCallback`：
```csharp
private EventCallback<ChangeEvent<int>> _totalDaysCallback;
// OnEnable: _totalDaysCallback = evt => UpdateTotalDaysLabel(evt.newValue);
//           _totalDaysSlider.RegisterValueChangedCallback(_totalDaysCallback);
// OnDisable: _totalDaysSlider.UnregisterValueChangedCallback(_totalDaysCallback);
```

### P2-1 · PausePanel.OnQuitClicked 退出到主菜单时没 DisableInput

**位置**：`PausePanel.OnQuitClicked`

**现象**：
```csharp
private void OnQuitClicked()
{
    SaveManager.Instance.Save(slotId);
    Time.timeScale = 1f;
    SceneManager.LoadScene(mainMenuSceneName);  // ← 没 DisableInput
}
```

对比 `MainMenuController.LoadGameScene`：
```csharp
InputManager.Instance.DisableInput();  // ← 进游戏场景前有禁用
Time.timeScale = 1f;
SceneManager.LoadScene(gameSceneName);
```

**后果**：回主菜单后 InputManager 还开着，玩家按移动键会触发 `PlayerMoveEvent`（虽然主菜单没 PlayerInputHandler，但 InputManager.OnMove 里会打 log）。

**修复**：`OnQuitClicked` 里 `SceneManager.LoadScene` 前加 `InputManager.Instance.DisableInput();`。

### P2-2 · Debug.Log 滥用（性能问题）

**位置**：
- `PlayerInputHandler.Update`：有移动输入时**每帧**打 `Debug.Log`
- `InputManager.OnMove`：每次移动输入都打 log
- `PausePanel.SetPanelVisible`：每次调用都打 log

**后果**：Release build 里 `Debug.Log` 仍有开销（字符串拼接 + 文件 I/O），按住移动键时每帧打 log 会**严重卡顿**。

**修复**：
- `PlayerInputHandler.Update` 的 log 删除或改 `[Conditional("UNITY_EDITOR")]`
- `InputManager.OnMove` 的 log 同上
- `PausePanel.SetPanelVisible` 的 log 删除

## 4. 场景预置单位检查（阿铁重点关注）

### 读档路径（ContinueFromSave）

```
GameBootstrap.InitializeScene
  └─ ContinueFromSave()
       ├─ RulerController.Instance.DestroyAllSceneUnits()  ← 清理所有场景预置 UnitController ✓
       ├─ SaveManager.Instance.Load(slotId)
       │    └─ SpawnFromSave 重建单位（HasSaveable 去重）✓
       └─ RulerController.Instance.BindExistingMonarch()  ← 绑定恢复的君主 ✓
```

**结论**：读档路径的场景预置单位清理**正确**，R2 修复有效。不会出现双份。

### 新建路径（StartNewGame）

```
GameBootstrap.InitializeScene
  └─ StartNewGame()
       ├─ WorldManager.ApplyConfig(...)
       ├─ RulerController.SpawnMonarch()
       │    └─ Step 1: RemoveDuplicateMonarchs()  ← 清理重复君主 ✓
       │    └─ Step 3: FindExistingMonarch()      ← 查找已有君主 ✓
       │    └─ Step 4: 只在没找到时才 SpawnUnit   ✓
       └─ SaveManager.Save(slotId)
```

**结论**：新建路径的场景预置单位处理**正确**，不会创建双份君主。

### 跨场景回归路径（问题所在）

```
GameScene → PausePanel.OnQuitClicked → LoadScene(MainMenuScene)
  → 旧场景 UnitController 被 Unity 销毁
  → _saveables / _aliveUnits 不清理 ← P0-1 问题在这里
  → MainMenuScene → OnSaveSlotSelected → LoadScene(GameScene)
  → GameBootstrap.Awake 不清理 ← P0-1 问题在这里
  → 新场景里 _aliveUnits 混有旧幽灵 + 新单位
```

**结论**：单次新建/读档不会出问题，但**反复进出场景**会累积幽灵引用。

## 5. 一句话总结

场景预置单位清理（R2）和存档去重（R3）都正常工作，新建/读档不会出现双份。**唯一严重问题是 R6 跨场景清理回归**——GameBootstrap 重构时丢了 `UnitRegistry.Clear()` 和 `CleanupDestroyedSaveables()` 两行调用，导致反复进出场景时内存泄漏 + 幽灵引用。加回这两行即可。UI 面板的按钮重复注册是次要问题，但 4 个面板都有，建议统一修。
