## 1. 引擎启动（CoreBootstrap）

### 1.1 执行时序

```
Unity 加载场景（MainMenuScene 或 GameScene）
  │
  ▼
CoreBootstrap.Awake  [DefaultExecutionOrder(-200)]
  │  两个场景都挂此脚本
  │
  ├─ 访问 GameStateManager.Instance
  │    └─ Singleton<T>.Instance getter：_instance==null → FindObjectOfType → 没找到 → 自动创建
  │         └─ new GameObject("[Singleton] GameStateManager") + AddComponent<GameStateManager>
  │              └─ GameStateManager.Awake  [同步执行，无视 ExecutionOrder]
  │                   ├─ base.Awake() → _instance = this, DontDestroyOnLoad(gameObject)
  │                   └─ CurrentState = GameState.Booting
  │
  ├─ 访问 InputManager.Instance  → 同上自动创建
  │    └─ InputManager.Awake  [同步]
  │         ├─ base.Awake() → _instance = this, DontDestroyOnLoad
  │         ├─ 检查 _instance != this（防重复）→ 如果是重复实例 return，不创建 GameInput
  │         ├─ _inputActions = new GameInput()
  │         ├─ 订阅 Input System：move.performed/canceled, fastmove.performed/canceled, esc.performed
  │         └─ _inputActions.Disable()  ← 默认禁用，等 GameBootstrap 切到 Playing 再开
  │
  ├─ 访问 SaveManager.Instance  → 同上自动创建
  │    └─ SaveManager.Awake  [DefaultExecutionOrder(-90)，但被同步触发]
  │         ├─ base.Awake() → _instance = this, DontDestroyOnLoad
  │         └─ LoadSettings()  ← 读 persistentDataPath/settings.json
  │
  ├─ 访问 WorldManager.Instance  → 同上自动创建
  │    └─ WorldManager.Awake  [同步]
  │         ├─ base.Awake() → _instance = this, DontDestroyOnLoad
  │         └─ SaveManager.Instance.RegisterSaveable(this)  ← 注册 WorldManager 为 ISaveable
  │
  └─ if (setBootingOnAwake && _isFirstCoreBootstrap)
       ├─ GameStateManager.Instance.SetState(GameState.Booting)
       │    └─ EventBus.Publish(GameStateChangedEvent(Boot, Booting))
       │         ⚠️ 此时 SaveManager.Start 还没执行（Start 在所有 Awake 之后）
       │            SaveManager 还没订阅 GameStateChangedEvent → EventBus 打"无订阅者"警告
       └─ _isFirstCoreBootstrap = false  ← 静态字段，跨场景不重置
```

### 1.2 关键点

| 要点 | 说明 |
|------|------|
| Singleton 自动创建 | `Singleton<T>.Instance` getter 在 `_instance==null` 时会 `new GameObject + AddComponent`，**同步触发 Awake**，无视 ExecutionOrder |
| R1 监控 | 自动创建时打 `Debug.LogWarning`，开发时看到这个 Warning 说明有代码提前访问了未放置的 Singleton |
| DontDestroyOnLoad | 所有 Singleton 的 `base.Awake()` 调 `DontDestroyOnLoad(gameObject)`，跨场景保留 |
| `_isFirstCoreBootstrap` | 静态字段，跨场景不重置。只有第一次进场景时设 Booting，后续场景切换不覆盖 |
| SaveManager.Start 延后 | SaveManager.Awake 期间不订阅事件，Start 里才订阅 `TimeDayChangedEvent` + `GameStateChangedEvent`。CoreBootstrap.Awake 发布的 `GameStateChangedEvent` 会被丢弃 |

### 1.3 此阶段被激活的模块

| 模块 | 激活方式 | 职责 |
|------|----------|------|
| GameStateManager | CoreBootstrap 显式访问 Instance | 全局状态机 |
| InputManager | CoreBootstrap 显式访问 Instance | 输入接收（默认禁用） |
| SaveManager | CoreBootstrap 显式访问 Instance | 存档读写 |
| WorldManager | CoreBootstrap 显式访问 Instance | 世界配置（种子/难度/总天数） |

### 1.4 此阶段未被激活的模块

| 模块 | 何时激活 | 触发点 |
|------|----------|--------|
| UnitDataManager | GameBootstrap.Start 首次访问 `Instance.IsInitialized` | `GameBootstrap.Start` |
| UnitFactory | GameBootstrap.Awake 首次访问 `Instance.RegisterSpawner` | `GameBootstrap.Awake` |
| UnitRegistry | 首次访问 `Instance`（读档时 `DestroyAllSceneUnits` 或新建时 `SpawnMonarch`） | `GameBootstrap.InitializeScene` |
| TimeManager | 首次访问 `Instance`（`GameBootstrap.InitializeScene` 里读时间） | `GameBootstrap.InitializeScene` |
| RulerController | 首次访问 `Instance`（`GameBootstrap.StartNewGame` 或 `ContinueFromSave`） | `GameBootstrap.InitializeScene` |

## 2. 主菜单场景（MainMenuScene）

### 2.1 启动到主菜单

```
所有 Awake 执行完毕后
  │
  ▼
所有 Start 执行
  │
  ├─ SaveManager.Start
  │    ├─ EventBus.Subscribe<TimeDayChangedEvent>(OnDayChanged)
  │    └─ EventBus.Subscribe<GameStateChangedEvent>(OnGameStateChanged)
  │
  └─ MainMenuController.Start
       ├─ GameStateManager.Instance.SetState(GameState.Splash)
       │    └─ EventBus.Publish(GameStateChangedEvent(Booting, Splash))
       │         └─ 此时 SaveManager 已订阅 → OnGameStateChanged 被调用
       │            （GameOver 才处理，Splash 无反应）
       └─ ShowPanel(Panel.Splash)
            ├─ splashRoot.SetActive(true)，其他面板 SetActive(false)
            ├─ GameStateManager.Instance.SetState(GameState.Splash)  ← 重复设，被 if 去重
            └─ SplashPanel.OnEnable 被调用（因为 splashRoot 从 inactive→active）
                 ├─ FindObjectOfType<MainMenuController>()
                 └─ 初始化闪烁状态
```

### 2.2 面板切换机制

`MainMenuController.ShowPanel(Panel panel)` 是唯一的面板切换入口：

```
ShowPanel(panel)
  ├─ _currentPanel = panel
  ├─ splashRoot.SetActive(panel == Panel.Splash)       → 触发 OnEnable/OnDisable
  ├─ mainMenuRoot.SetActive(panel == Panel.MainMenu)   → 触发 OnEnable/OnDisable
  ├─ saveSlotsRoot.SetActive(panel == Panel.SaveSlots) → 触发 OnEnable/OnDisable
  ├─ characterCreationRoot.SetActive(panel == Panel.CharacterCreation) → 触发 OnEnable/OnDisable
  └─ GameStateManager.Instance.SetState(对应状态)
       └─ EventBus.Publish(GameStateChangedEvent)
```

⚠️ **已知问题**：`PausePanel` / `GameOverPanel` / `SaveSlotPanel` / `CharacterCreationPanel` 的 `OnEnable` 里 `clicked +=` 没有防重复，反复切换面板会导致按钮回调注册多次。详见[排查指南 6.5](#65-按钮点击触发多次)。

### 2.3 新建游戏入口

```
MainMenuPanel "新建游戏" 按钮
  └─ MainMenuController.OnNewGameClicked()
       ├─ 检查是否有空槽位（HasAnySave + HasEmptySlot）
       └─ ShowPanel(Panel.CharacterCreation)
            └─ CharacterCreationPanel.OnEnable
                 ├─ FindObjectOfType<MainMenuController>()
                 ├─ 查询 UI 元素（name/difficulty/slot/slider）
                 ├─ 设置默认值（难度=Normal, 天数=60）
                 ├─ RegisterValueChangedCallback（难度联动天数、槽位占用检查）
                 └─ 绑定按钮（creation-back / creation-confirm）

玩家填完参数点"开始游戏"
  └─ CharacterCreationPanel.OnConfirmClicked()
       ├─ 校验 slotGroup.value >= 0
       ├─ NewGameConfig.CreateWithDifficulty(difficulty, name, slotId)
       │    └─ 按难度填充 startGold/Stone/Wood/Food + totalDays + 随机 mapSeed
       └─ _controller.OnCharacterCreateConfirmed(config)
            └─ MainMenuController.OnCharacterCreateConfirmed(config)
                 ├─ GameSceneEntrance.SetNewGame(config)
                 │    ├─ CurrentMode = Mode.NewGame
                 │    ├─ NewGameConfig = config
                 │    └─ LoadSlotId = config.selectedSlotId
                 └─ LoadGameScene()
                      ├─ InputManager.Instance.DisableInput()
                      ├─ Time.timeScale = 1f
                      └─ SceneManager.LoadScene(gameSceneName)
```

### 2.4 读档入口

```
MainMenuPanel "继续游戏" 按钮
  └─ MainMenuController.OnContinueClicked()
       └─ ShowPanel(Panel.SaveSlots)
            └─ SaveSlotPanel.OnEnable
                 ├─ FindObjectOfType<MainMenuController>()
                 ├─ 绑定 slots-back-button
                 └─ RebuildSlots()
                      ├─ _slotsList.Clear()
                      └─ for each slot_1/2/3:
                           ├─ SaveManager.Instance.GetSaveMeta(slotId)  ← 读 JSON 头部
                           ├─ new SaveSlotItem(slotId, meta)  ← 用 summary 显示信息
                           ├─ 创建"继续/新建到此处"按钮
                           └─ 创建"删除"按钮

玩家点存档槽的"继续"按钮
  └─ MainMenuController.OnSaveSlotSelected(slotId)
       ├─ SaveManager.Instance.HasSave(slotId) 校验
       ├─ GameSceneEntrance.SetContinue(slotId)
       │    ├─ CurrentMode = Mode.ContinueGame
       │    ├─ NewGameConfig = null
       │    └─ LoadSlotId = slotId
       └─ LoadGameScene()  ← 同 2.3
```

## 3. 新建游戏全链路

### 3.1 GameBootstrap.Awake [DefaultExecutionOrder(-100)]

```
GameBootstrap.Awake
  ├─ SaveManager.Instance.RegisterSpawner(UnitFactory.Instance)
  │    ├─ 首次访问 UnitFactory.Instance → 自动创建
  │    │    └─ UnitFactory.Awake  [同步]
  │    │         └─ base.Awake() → _instance = this, DontDestroyOnLoad
  │    └─ SaveManager._spawners.Add(UnitFactory)
  │
  └─ EventBus.Subscribe<UnitDataLoadedEvent>(OnDataLoaded)
```

⚠️ **缺失**：此处**没有**调用 `UnitRegistry.Clear()` 和 `SaveManager.CleanupDestroyedSaveables()`。跨场景回归时会累积幽灵引用。详见[排查指南 6.3](#63-内存泄漏幽灵引用)。

### 3.2 GameBootstrap.Start

```
GameBootstrap.Start
  ├─ GameStateManager.Instance.SetState(GameState.Loading)
  │    └─ EventBus.Publish(GameStateChangedEvent(Splash/Booting, Loading))
  │         ├─ SaveManager.OnGameStateChanged（非 GameOver，无操作）
  │         └─ LoadingPanel.OnGameStateChanged → SetPanelVisible(true)（如果 GameScene 挂了 LoadingPanel）
  │
  └─ if (UnitDataManager.Instance.IsInitialized)
       │  ├─ 首次访问 UnitDataManager.Instance → 自动创建
       │  │    └─ UnitDataManager.Awake  [同步]
       │  │         ├─ base.Awake() → _instance = this, DontDestroyOnLoad
       │  │         └─ LoadAll()
       │  │              ├─ Resources.LoadAll<UnitData>("UnitData")
       │  │              ├─ 缓存到 _dataCache
       │  │              ├─ IsInitialized = true
       │  │              └─ EventBus.Publish(UnitDataLoadedEvent(true, count))
       │  │                   ⚠️ 此时 GameBootstrap 刚 Subscribe 完，能收到
       │  │                   但下面又手动调了一次 OnDataLoaded（被 _sceneInitialized 去重）
       │
       └─ OnDataLoaded(new UnitDataLoadedEvent(true, count))
            └─ 见 3.3
```

### 3.3 InitializeScene → StartNewGame

```
OnDataLoaded(evt)
  ├─ _sceneInitialized 防重复检查
  ├─ GameStateManager.Instance.SetState(GameState.Ready)
  └─ InitializeScene()

InitializeScene()
  ├─ UnitFactory.Instance.PreloadAll()
  │    └─ Resources.LoadAll<GameObject>("UnitPrefabs") → 缓存到 _prefabCache
  │
  ├─ GameSceneEntrance.CurrentMode == NewGame → StartNewGame()
  │
  ├─ TimeManager 时间就绪日志（首次访问 TimeManager.Instance → 自动创建）
  │    └─ TimeManager.Awake [同步]
  │         ├─ base.Awake() → DontDestroyOnLoad
  │         ├─ 初始化 CurrentDay/Season/TimeOfDay/Phase
  │         └─ SaveManager.Instance.RegisterSaveable(this)
  │
  ├─ InputManager.Instance.EnableInput()
  │    └─ _inputActions.Enable()
  │
  ├─ GameStateManager.Instance.SetState(GameState.Playing)
  │    └─ EventBus.Publish(GameStateChangedEvent(Ready, Playing))
  │         ├─ SaveManager.OnGameStateChanged（非 GameOver，无操作）
  │         ├─ PausePanel.OnGameStateChanged → SetPanelVisible(false)
  │         └─ GameOverPanel.OnGameStateChanged → SetPanelVisible(false)
  │
  └─ GameSceneEntrance.Clear()
       └─ CurrentMode = NewGame, NewGameConfig = null, LoadSlotId = null
```

### 3.4 StartNewGame 详细

```
StartNewGame()
  ├─ var config = GameSceneEntrance.NewGameConfig
  │
  ├─ if (config != null):
  │    ├─ WorldManager.Instance.ApplyConfig(mapSeed, difficulty, totalDays)
  │    │    └─ 设置 MapSeed / Difficulty / TotalDays
  │    │       TODO: GenerateWorld(MapSeed)（暂未实现）
  │    │
  │    ├─ RulerController.Instance.SetRulerName(config.rulerName)
  │    │    └─ 首次访问 RulerController.Instance → 自动创建空 GameObject
  │    │         └─ RulerController.Awake [同步]
  │    │              ├─ _instance == null → _instance = this, _isRealSingleton = true
  │    │              ├─ DontDestroyOnLoad(gameObject)
  │    │              ├─ TryLoadRulerData()
  │    │              │    ├─ rulerData == null（空对象无 SerializeField）
  │    │              │    └─ UnitDataManager.Instance.IsInitialized == true
  │    │              │         └─ FetchRulerDataFromManager()
  │    │              │              ├─ UnitDataManager.GetData(Human_Player, Ruler)
  │    │              │              ├─ rulerData = data as RulerData
  │    │              │              └─ ApplyRulerData() → Gold/Stone/Wood/Food = 资产默认值
  │    │              ├─ EventBus.Subscribe<UnitDataLoadedEvent>(OnUnitDataLoaded)
  │    │              ├─ EventBus.Subscribe<UnitDiedEvent>(OnUnitDied)
  │    │              └─ SaveManager.Instance.RegisterSaveable(this)
  │    │
  │    ├─ RulerController.Instance.ApplyStartResources(gold, stone, wood, food)
  │    │    └─ 覆盖 ApplyRulerData 设的默认值，用玩家配置的资源
  │    │
  │    └─ TimeManager.Instance.ApplyConfig(totalDays)
  │         └─ SetTotalDays(totalDays)
  │
  ├─ RulerController.Instance.SpawnMonarch()  ← 见 3.5
  │
  └─ 自动存初始档：
       ├─ slotId = config.selectedSlotId ?? "slot_1"
       ├─ if (!SaveManager.Instance.HasSave(slotId))
       │    └─ SaveManager.Instance.Save(slotId)
       │         ├─ CurrentSlotId = slotId
       │         ├─ root.summary = BuildSummary()  ← 从 RulerController/TimeManager/WorldManager 抓字段
       │         ├─ 遍历 _saveables 收集 SaveState
       │         ├─ JsonUtility.ToJson(root)
       │         ├─ 原子写入（.tmp → 替换）
       │         └─ EventBus.Publish(GameSavedEvent(slotId, true))
       └─ GameSceneEntrance.Clear()
```

### 3.5 SpawnMonarch 4 步防御

```
RulerController.SpawnMonarch()
  │
  ├─ Step 0: 验证 monarchUnit
  │    ├─ if (monarchUnit != null && monarchUnit.gameObject != null) → return（已有君主）
  │    └─ if (monarchUnit != null && gameObject == null) → monarchUnit = null（清理悬空引用）
  │
  ├─ Step 1: RemoveDuplicateMonarchs()
  │    ├─ FindObjectsOfType<UnitController>()
  │    ├─ 识别君主（PlayerInputHandler 或 Data.faction==Human_Player && occupation==Ruler）
  │    ├─ 保留第一个，其余：
  │    │    ├─ SaveManager.Instance.UnregisterSaveable(unit)
  │    │    ├─ UnitRegistry.Instance.Unregister(unit)
  │    │    │    └─ 首次访问 UnitRegistry.Instance → 自动创建
  │    │    └─ Destroy(unit.gameObject)
  │    └─ 返回清理数量
  │
  ├─ Step 2: 确保 rulerData
  │    └─ if (rulerData == null) → FetchRulerDataFromManager()
  │
  ├─ Step 3: FindExistingMonarch()
  │    ├─ FindObjectsOfType<UnitController>()
  │    ├─ 优先级 1: Data 已初始化 + faction=Human_Player + occupation=Ruler
  │    ├─ 优先级 2: Data 已初始化 + 有 PlayerInputHandler
  │    ├─ 优先级 3: Data==null + 有 PlayerInputHandler（场景手动放置未初始化）
  │    └─ 返回找到的 UnitController 或 null
  │
  ├─ if (monarchUnit != null):
  │    ├─ if (Data == null) → monarchUnit.Initialize(rulerData)
  │    │    ├─ Data = rulerData
  │    │    ├─ MaxHp/Attack/Defense/Speed = 资产值
  │    │    ├─ CurrentHp = MaxHp
  │    │    ├─ UnitRegistry.Instance.Register(this)
  │    │    ├─ SaveId = "Unit_Human_Player_Ruler_{GUID}"
  │    │    ├─ SaveManager.Instance.RegisterSaveable(this)
  │    │    └─ EventBus.Publish(UnitSpawnedEvent(this))
  │    │         └─ TopLeftHUD.OnUnitSpawned → 检查 faction+occupation → TryBindMonarch()
  │    └─ return（绑定完成）
  │
  └─ Step 4: 场景中确实没有君主 → 代码兜底创建
       ├─ UnitFactory.Instance.PreloadAll()
       ├─ UnitFactory.Instance.SpawnUnit(rulerData, spawnPosition)
       │    ├─ key = "Human_Player_Ruler"
       │    ├─ _prefabCache.TryGetValue(key, out prefab)
       │    ├─ Instantiate(prefab, position, Quaternion.identity)
       │    ├─ instance.name = key
       │    ├─ controller.Initialize(data)  ← 同上 Step 3 的 Initialize
       │    └─ 返回 GameObject
       └─ monarchUnit = rulerGo.GetComponent<UnitController>()
```

## 4. 读档全链路

### 4.1 ContinueFromSave

```
InitializeScene()
  ├─ UnitFactory.Instance.PreloadAll()
  │
  ├─ GameSceneEntrance.CurrentMode == ContinueGame → ContinueFromSave()
  │
  ├─ ...（后续同新建：时间日志 + EnableInput + SetState(Playing) + Clear）

ContinueFromSave()
  ├─ ① RulerController.Instance.DestroyAllSceneUnits()
  │    ├─ 首次访问 RulerController.Instance → 自动创建（同 3.4）
  │    ├─ FindObjectsOfType<UnitController>()
  │    └─ 对每个 UnitController:
  │         ├─ SaveManager.Instance.UnregisterSaveable(unit)  ← SaveId 可能为 null，内部已防御
  │         ├─ UnitRegistry.Instance.Unregister(unit)  ← 首次访问 UnitRegistry.Instance → 自动创建
  │         ├─ Destroy(unit.gameObject)
  │         └─ monarchUnit = null  ← 清除引用
  │
  ├─ ② SaveManager.Instance.Load(slotId)
  │    └─ 见 4.2
  │
  └─ ③ RulerController.Instance.BindExistingMonarch()
       ├─ if (monarchUnit != null && gameObject != null) → return（已绑定）
       ├─ FindExistingMonarch()  ← 同 3.5 Step 3
       └─ monarchUnit = 找到的单位
```

### 4.2 SaveManager.Load 三阶段

```
SaveManager.Load(slotId)
  ├─ 读文件 → JsonUtility.FromJson<GameSaveRoot>
  ├─ 检查 isFinished（拒绝加载死档）
  ├─ 检查 saveVersion（拒绝高版本）
  ├─ CurrentSlotId = slotId
  │
  ├─ 阶段 1: DistributePayloads(root.modules, SaveLoadPhase.Global)
  │    └─ 遍历 modules，对 phase==Global 的条目：
  │         ├─ _saveables.TryGetValue(saveId, out saveable)
  │         └─ saveable.LoadState(payload)
  │
  │    涉及的模块：
  │    ├─ WorldManager.LoadState  ← 恢复 mapSeed/difficulty/totalDays
  │    ├─ TimeManager.LoadState   ← 恢复 currentDay/timeOfDay/season/phase + 配置参数
  │    └─ RulerController.LoadState ← 恢复 rulerName/gold/stone/wood/food
  │         └─ 不在这里 SpawnMonarch（君主走 UnitController ISaveable 流程）
  │
  ├─ 阶段 1.5: 场景对象重建（SpawnFromSave）
  │    └─ 遍历 modules，对 phase==Scene 的条目：
  │         └─ 遍历 _spawners，匹配 SaveIdPrefix：
  │              └─ UnitFactory.SpawnFromSave(entry)
  │                   ├─ 检查 typeName == UnitSaveData
  │                   ├─ R3 去重：if (SaveManager.Instance.HasSaveable(entry.saveId)) → 跳过
  │                   ├─ JsonUtility.FromJson<UnitSaveData>(entry.json)
  │                   ├─ UnitDataManager.Instance.GetData(faction, occupation) → 获取配置
  │                   ├─ SpawnUnit(config, pos)
  │                   │    ├─ Instantiate(prefab, pos, Quaternion.identity)
  │                   │    └─ controller.Initialize(data)
  │                   │         ├─ Data = config
  │                   │         ├─ MaxHp/Attack/Defense/Speed = 配置值
  │                   │         ├─ UnitRegistry.Instance.Register(this)
  │                   │         ├─ SaveId = "Unit_{faction}_{occupation}_{新GUID}"
  │                   │         ├─ SaveManager.Instance.RegisterSaveable(this)
  │                   │         └─ EventBus.Publish(UnitSpawnedEvent(this))
  │                   └─ controller.OverrideSaveId(entry.saveId)
  │                        ├─ oldId = SaveId（新 GUID）
  │                        ├─ SaveId = entry.saveId（存档 ID）
  │                        └─ SaveManager.Instance.ChangeSaveId(oldId, entry.saveId, this)
  │                             ├─ _saveables.Remove(oldId)
  │                             └─ _saveables.Add(newId, this)
  │
  ├─ 阶段 2: DistributePayloads(root.modules, SaveLoadPhase.Scene)
  │    └─ 遍历 modules，对 phase==Scene 的条目：
  │         └─ saveable.LoadState(payload)
  │
  │    涉及的模块：
  │    └─ UnitController.LoadState  ← 恢复 currentHp/maxHp/attack/defense/speed
  │         （位置已在 SpawnFromSave 时由 SpawnUnit 设置）
  │
  └─ EventBus.Publish(GameLoadedEvent(slotId, true))
```

## 5. 运行时关键事件

### 5.1 天推进

```
TimeManager.Update  [仅 GameState.Playing 时]
  └─ AdvanceTime(Time.deltaTime)
       ├─ _dayTimer += delta
       ├─ if (_dayTimer >= secondsPerDay):
       │    ├─ _dayTimer -= secondsPerDay
       │    └─ AdvanceDay()
       │         ├─ CurrentDay++
       │         ├─ 计算新季节，如果变化：
       │         │    └─ EventBus.Publish(SeasonChangedEvent(oldSeason, newSeason))
       │         │         ├─ TopLeftHUD.OnSeasonChanged → UpdateSeasonText
       │         │         └─ 其他订阅者
       │         └─ EventBus.Publish(TimeDayChangedEvent(oldDay, newDay, currentSeason))
       │              ├─ SaveManager.OnDayChanged
       │              │    ├─ _daysSinceLastAutoSave++
       │              │    └─ if (_daysSinceLastAutoSave >= AutoSaveIntervalDays):
       │              │         ├─ Save(CurrentSlotId)  ← 自动存档
       │              │         └─ _daysSinceLastAutoSave = 0
       │              └─ TopLeftHUD.OnDayChanged → UpdateDayText + UpdateSeasonText
       │
       ├─ CurrentTimeOfDay = (_dayTimer / secondsPerDay) * 24f
       ├─ 计算新时段，如果变化：
       │    └─ EventBus.Publish(TimePhaseChangedEvent(oldPhase, newPhase))
       └─ CurrentHour = FloorToInt(CurrentTimeOfDay)
```

### 5.2 君主死亡 → GameOver

```
UnitController.TakeDamage(rawDamage, source)
  ├─ actualDamage = Max(1, rawDamage - Defense)
  ├─ CurrentHp = Max(0, CurrentHp - actualDamage)
  ├─ EventBus.Publish(UnitDamagedEvent(this, source, actualDamage))
  ├─ EventBus.Publish(UnitHpChangedEvent(this, oldHp, CurrentHp, MaxHp))
  │    └─ TopLeftHUD.OnHpChanged → UpdateHpBar
  └─ if (CurrentHp <= 0) → Die()

UnitController.Die()
  ├─ SaveManager.Instance.UnregisterSaveable(this)  ← 先注销，防 Save 抓到已销毁对象
  ├─ EventBus.Publish(UnitDiedEvent(this))
  │    ├─ RulerController.OnUnitDied
  │    │    └─ if (evt.Unit == monarchUnit) → OnMonarchDied()
  │    │         ├─ monarchUnit = null
  │    │         └─ GameStateManager.Instance.SetState(GameState.GameOver)
  │    │              └─ EventBus.Publish(GameStateChangedEvent(Playing, GameOver))
  │    │                   ├─ SaveManager.OnGameStateChanged
  │    │                   │    └─ MarkCurrentSaveFinished()  ← 存档标记 isFinished=true
  │    │                   ├─ PausePanel.OnGameStateChanged → SetPanelVisible(false)
  │    │                   └─ GameOverPanel.OnGameStateChanged → Show()
  │    │                        ├─ Time.timeScale = 0f
  │    │                        ├─ SetPanelVisible(true)
  │    │                        └─ PopulateStats()  ← 读 TimeManager/RulerController 显示结算
  │    └─ TopLeftHUD.OnUnitDied → UpdateHpBar(0) + _monarch = null
  │
  ├─ UnitRegistry.Instance.Unregister(this)
  └─ Destroy(gameObject)
```

### 5.3 ESC 暂停/恢复

```
玩家按 ESC
  └─ InputManager.OnEsc(CallbackContext)
       └─ EventBus.Publish(EscapePressedEvent(currentState))
            └─ PausePanel.OnEscapePressed(evt)
                 ├─ case Playing: → Pause()
                 │    ├─ SetPanelVisible(true)
                 │    ├─ Time.timeScale = 0f
                 │    └─ GameStateManager.Instance.SetState(GameState.Paused)
                 │         └─ EventBus.Publish(GameStateChangedEvent(Playing, Paused))
                 │              └─ TimeManager.Update 检查 CurrentState != Playing → 停止推进时间
                 │
                 └─ case Paused: → Resume()
                      ├─ SetPanelVisible(false)
                      ├─ Time.timeScale = 1f
                      └─ GameStateManager.Instance.SetState(GameState.Playing)
```

### 5.4 暂停退出回主菜单

```
PausePanel.OnQuitClicked()
  ├─ SaveManager.Instance.Save(CurrentSlotId ?? "slot_1")  ← 退出前自动存档
  ├─ Time.timeScale = 1f
  └─ SceneManager.LoadScene(mainMenuSceneName)
       │
       │  ⚠️ 此时 GameScene 的所有非 DontDestroyOnLoad 对象被 Unity 销毁
       │  ⚠️ 但 _saveables / _aliveUnits 里的 C# 引用不会被清理（R6 回归）
       │
       ▼
MainMenuScene 加载
  └─ CoreBootstrap.Awake（_isFirstCoreBootstrap 已是 false，不设 Booting）
       └─ Singleton 已存在，不重新创建
```

## 6. 排查指南

### 6.1 君主没创建

| 症状 | 排查路径 |
|------|----------|
| `MonarchUnit == null` | 1. 查 Console 有无 `[RulerController] 没有君主 RulerData` → `Resources/UnitData/` 下缺 `Human_Player_Ruler.asset` |
| | 2. 查 Console 有无 `[UnitFactory] 找不到 Prefab: Human_Player_Ruler` → `Resources/UnitPrefabs/` 下缺 prefab |
| | 3. 查 `SpawnMonarch` 走到哪一步：Step 0 已有？Step 1 清重？Step 3 找到？Step 4 创建？ |
| | 4. 查 RulerController.Instance 是否是空对象（R5）：Console 有无 `[RulerController] 检测到重复实例` |
| 读档后 MonarchUnit == null | 1. 查 `BindExistingMonarch` 是否被调用（GameBootstrap.ContinueFromSave 第 ③ 步） |
| | 2. 查 `SaveManager.Load` 阶段 1.5 是否创建了君主（存档里有没有 `Unit_Human_Player_Ruler_*` 条目） |
| | 3. 查 `FindExistingMonarch` 3 级优先级是否都没找到 |

### 6.2 读档后双份单位

| 症状 | 排查路径 |
|------|----------|
| 场景里有两个相同单位 | 1. 查 `DestroyAllSceneUnits` 是否被调用（GameBootstrap.ContinueFromSave 第 ① 步） |
| | 2. 查存档 JSON 里有没有重复 saveId 的条目（`root.modules` 里同 saveId 出现两次） |
| | 3. 查 `SpawnFromSave` 的 R3 去重是否生效：Console 有无 `SaveId '...' 已存在，跳过重复创建` |

### 6.3 内存泄漏/幽灵引用

| 症状 | 排查路径 |
|------|----------|
| `MissingReferenceException` 访问已销毁单位 | 1. 查是否反复进出场景（GameScene → 主菜单 → GameScene） |
| | 2. 查 `GameBootstrap.Awake` 是否调了 `UnitRegistry.Clear()` + `SaveManager.CleanupDestroyedSaveables()` ← **当前缺失（P0-1）** |
| | 3. 查 `_saveables` / `_aliveUnits` 是否有已销毁的 MonoBehaviour 引用 |
| `_aliveUnits.Count` 持续增长 | 同上，跨场景清理缺失 |

### 6.4 事件无订阅者警告

| 症状 | 排查路径 |
|------|----------|
| `[EventBus] 事件 XxxEvent 无订阅者` | 1. 查发布时机：是不是在订阅者 Awake 之前发布的（如 CoreBootstrap.Awake 发布的 GameStateChangedEvent） |
| | 2. 查订阅者是否正确订阅：用 `EventBus.HasSubscribers<XxxEvent>()` 检查 |
| | 3. 查订阅者的 MonoBehaviour 是否被场景销毁但没退订（检查 OnDestroy / OnDisable） |

### 6.5 按钮点击触发多次

| 症状 | 排查路径 |
|------|----------|
| 点一次按钮触发 N 次回调 | 1. 查对应面板的 `OnEnable` 是否每次都 `clicked +=`（PausePanel / GameOverPanel / SaveSlotPanel / CharacterCreationPanel） |
| | 2. 查 `MainMenuController.ShowPanel` 用 `SetActive` 切换面板的次数 |
| | 3. 修复：加 `OnDisable` 里 `clicked -=`，或 `OnEnable` 里加 `_buttonsBound` 检查 |

### 6.6 存档槽 UI 显示空值

| 症状 | 排查路径 |
|------|----------|
| 存档槽显示 `rulerName=null, currentDay=0` | 1. 查 `SaveManager.Save` 是否调了 `BuildSummary()`（已修复） |
| | 2. 查 `BuildSummary` 里各 Manager 的 Instance 是否为 null（try-catch 防御了，但会返回默认值） |
| | 3. 查存档 JSON 文件里 `summary` 字段是否有值 |

### 6.7 InputManager 自动创建副作用

| 症状 | 排查路径 |
|------|----------|
| Console 出现 `[InputManager] 场景中未找到实例，自动创建` | 1. 查是谁在 CoreBootstrap 之前访问了 `InputManager.Instance` |
| | 2. CoreBootstrap.Awake 应该是第一个访问 InputManager 的，如果有更早的访问者说明执行顺序有问题 |
| | 3. 自动创建的 InputManager 没有 Inspector 配置，GameInput 可能行为异常 |

### 7.4 ExecutionOrder 优先级表

| 优先级 | 脚本 | 所在场景 |
|--------|------|----------|
| -200 | CoreBootstrap | 两个场景都挂 |
| -100 | GameBootstrap | GameScene |
| -90 | SaveManager | 自动创建（DontDestroyOnLoad） |
| 无 | 其他所有脚本 | 各自场景 |

> **注意**：Singleton 自动创建会**同步触发** Awake，无视 ExecutionOrder。例如 CoreBootstrap.Awake [-200] 里访问 `SaveManager.Instance` → SaveManager.Awake 立即同步执行，不会等 -90 的顺序。ExecutionOrder 只对**场景里预先挂载**的脚本生效。
